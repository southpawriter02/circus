# FXEnableExtensionChangeWarning

Disable warning when changing file extensions.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.finder` |
| **Key** | `FXEnableExtensionChangeWarning` |
| **Type** | Boolean |
| **Default Value** | `true` |
| **Our Setting** | `false` |

## Description

Controls whether Finder shows a warning dialog when you change a file's extension (e.g., renaming `.txt` to `.md`).

## Values

| Value | Behavior |
|-------|----------|
| `true` | **Show warning** (default) — Dialog asks to confirm extension change |
| `false` | **No warning** — Extension changes applied silently |

## Our Configuration

**Set to**: `false` — Suppress warning for power users who understand extensions.

## Code Examples

```bash
# Read current value
defaults read com.apple.finder FXEnableExtensionChangeWarning

# Disable extension change warning
defaults write com.apple.finder FXEnableExtensionChangeWarning -bool false

# Enable warning (Apple default)
defaults write com.apple.finder FXEnableExtensionChangeWarning -bool true

# Delete (reset to default)
defaults delete com.apple.finder FXEnableExtensionChangeWarning

# Restart Finder to apply
killall Finder
```

## UI Location

No direct toggle. The warning dialog appears when renaming a file with a different extension.

## Note

Disabling this is recommended for developers who frequently rename files and understand the implications of changing extensions.

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Warning still showing | Restart Finder: `killall Finder` |
| Some extensions still warn | System-protected extensions may still warn |
| Accidentally changed extension | Use ⌘Z to undo rename |

## Compatibility

- **Finder**: macOS 10.5+
- **Applies to**: All file renaming operations

## Source File

**File**: [`defaults/interface/finder.sh`](../../../defaults/interface/finder.sh)
