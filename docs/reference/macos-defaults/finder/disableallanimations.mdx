# Finder Disable All Animations

Control whether Finder window animations are disabled for faster navigation.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.finder` |
| **Key** | `DisableAllAnimations` |
| **Type** | Boolean |
| **Default Value** | `false` |
| **Our Setting** | `false` |

## Description

Controls whether all Finder window animations are disabled. When enabled, Finder operations like opening folders, resizing windows, and Quick Look become instant instead of animated. This can make Finder feel significantly snappier, especially on older hardware or when working with many files. Power users who prioritize speed over visual polish often enable this setting.

## Values

| Value | Behavior |
|-------|----------|
| `false` | **Animations enabled** (default) â€” Smooth, polished transitions |
| `true` | **Animations disabled** â€” Instant, no-animation mode |

## Our Configuration

**Set to**: `false` â€” Keep animations for polished experience.

### Why Keep Animations (false)?

1. **Visual feedback** â€” Animations show what's happening
2. **Polished feel** â€” macOS design language maintained
3. **Easier to follow** â€” Transitions help track changes
4. **Modern hardware** â€” Animations are smooth on current Macs
5. **Default experience** â€” Consistent with expectations

### Why Disable Animations (true)?

- **Speed** â€” Instant response feels faster
- **Efficiency** â€” Less waiting during file operations
- **Focus** â€” No distracting motion
- **Old hardware** â€” Better performance on older Macs
- **Preference** â€” Some users prefer instant feedback

## Code Examples

```bash
# Read current value
defaults read com.apple.finder DisableAllAnimations

# Disable all Finder animations (for speed)
defaults write com.apple.finder DisableAllAnimations -bool true

# Enable Finder animations (default)
defaults write com.apple.finder DisableAllAnimations -bool false

# Delete (reset to default)
defaults delete com.apple.finder DisableAllAnimations

# Restart Finder to apply changes
killall Finder
```

### Related Animation Commands

```bash
# Disable Finder animations
defaults write com.apple.finder DisableAllAnimations -bool true

# Speed up window resize animation (system-wide)
defaults write NSGlobalDomain NSWindowResizeTime -float 0.001

# Disable window animations when opening/closing (system-wide)
defaults write NSGlobalDomain NSAutomaticWindowAnimationsEnabled -bool false

# Speed up Mission Control animations
defaults write com.apple.dock expose-animation-duration -float 0

# Speed up Dock show/hide
defaults write com.apple.dock autohide-time-modifier -float 0

# Apply all changes
killall Finder
killall Dock
```

## Related Settings

```bash
# Disable Finder animations (this setting)
defaults write com.apple.finder DisableAllAnimations -bool true

# Disable window opening animation
defaults write NSGlobalDomain NSAutomaticWindowAnimationsEnabled -bool false

# Speed up window resize
defaults write NSGlobalDomain NSWindowResizeTime -float 0.001

# Disable Quick Look animation
defaults write -g QLPanelAnimationDuration -float 0

# Disable Dock animations
defaults write com.apple.dock launchanim -bool false

# Disable folder spring-loading delay
defaults write NSGlobalDomain com.apple.springing.delay -float 0

# Restart affected services
killall Finder
killall Dock
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Setting not applying | Run `killall Finder` after changing |
| Some animations still occur | Use additional settings above for full effect |
| Finder crashes after change | Delete setting and restart Finder |
| Want partial animations | This is all-or-nothing; use system Reduce Motion instead |
| Miss animations, want them back | Set to false and restart Finder |
| Still feels slow | Check for indexing (Spotlight) or disk issues |

## Compatibility

- **macOS**: 10.7+ (Lion and later)
- **Note**: Requires `killall Finder` to apply changes
- **Note**: Setting persists across restarts
- **Note**: Does not affect other applications
- **Note**: Works alongside system-wide Reduce Motion setting
- **Note**: May interact with accessibility settings
