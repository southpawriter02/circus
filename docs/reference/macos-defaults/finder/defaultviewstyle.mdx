# Finder Default View Style

Set the default view style for new Finder windows.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.finder` |
| **Key** | `FXPreferredViewStyle` |
| **Type** | String |
| **Default Value** | `icnv` |
| **Our Setting** | `Nlsv` |

## Description

Controls the default view style used when opening new Finder windows. This determines how files and folders are displayed â€” as icons in a grid, as a detailed list, in columns, or in gallery format.

## Values

| Value | View Style | Description |
|-------|------------|-------------|
| `icnv` | **Icons** (default) | Grid of icons with names below |
| `Nlsv` | **List** | Detailed rows with columns |
| `clmv` | **Columns** | Hierarchical columns (NeXT-style) |
| `glyv` | **Gallery** | Large preview with thumbnails |

## Our Configuration

**Set to**: `Nlsv` â€” List view.

### Why List View?

1. **Information density** â€” See details at a glance
2. **Sortable columns** â€” Click to sort by date, size, kind
3. **Efficient** â€” Many files visible at once
4. **Hierarchical** â€” Expand folders in place
5. **Professional** â€” Developer/power user choice

### Why Icon View?

- Visual browsing (photos, design files)
- Spatial memory (remember file positions)
- Touch-friendly (larger targets)

### Why Column View?

- Deep folder navigation
- Path visualization
- NeXT/traditional Mac feel

### Why Gallery View?

- Photo/media browsing
- Large previews
- Presentation selection

## Code Examples

```bash
# Read current value
defaults read com.apple.finder FXPreferredViewStyle

# List view (recommended)
defaults write com.apple.finder FXPreferredViewStyle -string "Nlsv"

# Icon view (default)
defaults write com.apple.finder FXPreferredViewStyle -string "icnv"

# Column view
defaults write com.apple.finder FXPreferredViewStyle -string "clmv"

# Gallery view
defaults write com.apple.finder FXPreferredViewStyle -string "glyv"

# Restart Finder to apply
killall Finder
```
