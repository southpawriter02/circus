# Finder New Window Target Path

Set the default folder for new Finder windows.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.finder` |
| **Key** | `NewWindowTargetPath` |
| **Type** | String (file:// URL) |
| **Default Value** | `file:///System/Library/CoreServices/Finder.app/Contents/Resources/MyLibraries/myDocuments.cannedSearch/` |
| **Our Setting** | `file://${HOME}/` |

## Description

Specifies the exact folder path that opens when you create a new Finder window (âŒ˜N) or click the Finder icon in the Dock. This works in conjunction with `NewWindowTarget` to provide precise control over your default Finder location. Setting this to your home folder provides quick access to your personal files.

## Values

| Value | Location |
|-------|----------|
| `file://${HOME}/` | Home folder (~) |
| `file://${HOME}/Desktop/` | Desktop |
| `file://${HOME}/Documents/` | Documents folder |
| `file://${HOME}/Downloads/` | Downloads folder |
| `file:///` | Computer (root) |
| `file:///Applications/` | Applications folder |
| `file:///Users/` | Users folder |
| Custom path | Any valid folder as file:// URL |

## Our Configuration

**Set to**: `file://${HOME}/` â€” Open home folder in new windows.

### Why Use Home Folder?

1. **Central hub** â€” Access all personal folders
2. **Fast navigation** â€” One click to any location
3. **Predictable** â€” Always know where you start
4. **Developer-friendly** â€” Quick access to dotfiles
5. **Performance** â€” Faster than Recents search

### Why Use Other Locations?

| Location | Best For |
|----------|----------|
| Desktop | Quick file access, active projects |
| Documents | Document-centric workflows |
| Downloads | Frequent download management |
| Projects folder | Developers |
| Recents | Finding recent files (default) |

## Related Setting: NewWindowTarget

This setting works with `NewWindowTarget`:

```bash
# NewWindowTarget controls the type
# NewWindowTargetPath specifies the exact path

# For custom path:
defaults write com.apple.finder NewWindowTarget -string "PfLo"
defaults write com.apple.finder NewWindowTargetPath -string "file://${HOME}/"

# NewWindowTarget values:
# PfCm = Computer
# PfVo = Volume (disk)
# PfHm = Home folder
# PfDe = Desktop
# PfDo = Documents
# PfAF = All My Files
# PfLo = Other (uses NewWindowTargetPath)
```

## Code Examples

```bash
# Read current value
defaults read com.apple.finder NewWindowTargetPath

# Set to home folder
defaults write com.apple.finder NewWindowTarget -string "PfLo"
defaults write com.apple.finder NewWindowTargetPath -string "file://${HOME}/"

# Set to Desktop
defaults write com.apple.finder NewWindowTarget -string "PfDe"

# Set to Documents
defaults write com.apple.finder NewWindowTarget -string "PfDo"

# Set to custom folder (e.g., Projects)
defaults write com.apple.finder NewWindowTarget -string "PfLo"
defaults write com.apple.finder NewWindowTargetPath -string "file://${HOME}/Projects/"

# Delete (reset to default - Recents)
defaults delete com.apple.finder NewWindowTarget
defaults delete com.apple.finder NewWindowTargetPath

# Restart Finder to apply
killall Finder
```

## URL Encoding

For paths with spaces or special characters, use URL encoding:

| Character | Encoded |
|-----------|---------|
| Space | `%20` |
| & | `%26` |
| = | `%3D` |

Example:
```bash
# "My Projects" folder
defaults write com.apple.finder NewWindowTargetPath -string "file://${HOME}/My%20Projects/"
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Still opens Recents | Set both NewWindowTarget and NewWindowTargetPath |
| Path not found | Verify folder exists, check URL encoding |
| Changes not applying | Run `killall Finder` |
| iCloud path not working | Use full Mobile Documents path |

## Finder Window vs Finder Click

| Action | Behavior |
|--------|----------|
| **âŒ˜N** | Opens NewWindowTargetPath |
| **Click Finder in Dock** | Opens NewWindowTargetPath (if no windows open) |
| **Click Finder icon (window open)** | Brings existing window forward |
| **Double-click folder** | Opens that specific folder |
