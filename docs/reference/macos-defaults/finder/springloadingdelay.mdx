# Finder Spring Loading Delay

Control the delay before folders open when dragging files over them.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `NSGlobalDomain` |
| **Key** | `com.apple.springing.delay` |
| **Type** | Float (seconds) |
| **Default Value** | `0.5` |
| **Our Setting** | `0.3` |

## Description

Controls the amount of time you must hover over a folder while dragging before the folder automatically opens (springs open). This feature allows you to navigate through folder hierarchies without releasing the item you're dragging. A shorter delay makes navigation faster but may cause accidental folder openings; a longer delay is more deliberate but slower.

## Values

| Value | Speed | Description |
|-------|-------|-------------|
| `0.1` | Very Fast | Nearly instant, may cause accidents |
| `0.3` | Fast | Quick response, good for experienced users |
| `0.5` | Normal | Default macOS timing |
| `0.75` | Slow | More deliberate |
| `1.0` | Very Slow | Extra time to position |
| `1.5` | Slowest | Maximum deliberation |

## Our Configuration

**Set to**: `0.3` â€” Faster spring loading for efficient file management.

### Why Use Shorter Delay?

1. **Speed** â€” Navigate folders faster
2. **Productivity** â€” Less waiting while organizing files
3. **Flow** â€” Maintain momentum while dragging
4. **Efficiency** â€” Move files to deep folders quickly
5. **Power users** â€” Familiar with folder structure

### Why Use Longer Delay?

- Prevent accidental folder openings
- More time to position cursor precisely
- Less experienced with file navigation
- Complex folder structures
- Accessibility needs

## Code Examples

```bash
# Read current delay
defaults read NSGlobalDomain com.apple.springing.delay

# Set fast delay (0.3 seconds)
defaults write NSGlobalDomain com.apple.springing.delay -float 0.3

# Set default delay (0.5 seconds)
defaults write NSGlobalDomain com.apple.springing.delay -float 0.5

# Set slow delay (1.0 second)
defaults write NSGlobalDomain com.apple.springing.delay -float 1.0

# Delete (reset to default)
defaults delete NSGlobalDomain com.apple.springing.delay
```

Changes take effect immediately.

defaults write NSGlobalDomain com.apple.springing.delay -float 0.3

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Folders not opening | Check spring loading is enabled |
| Opens too fast | Increase delay value |
| Opens too slow | Decrease delay value |
| Accidental openings | Increase delay or use Space shortcut |
| Setting not applying | Value takes effect immediately |

## Recommended Configurations

| User Type | Delay | Reasoning |
|-----------|-------|-----------|
| Power user | 0.2-0.3s | Fast navigation, knows folder structure |
| Regular user | 0.5s | Default, balanced |
| New user | 0.7-1.0s | More time to position |
| Accessibility | 1.0-1.5s | Deliberate, no accidents |

## Disable Spring Loading

If you find spring loading annoying:

```bash
# Disable spring loading entirely
defaults write NSGlobalDomain com.apple.springing.enabled -bool false
```

Then folders only open when you drop files on them directly.

## Compatibility

- **macOS**: 10.0+ (original Mac OS X feature)
- **Note**: Takes effect immediately
- **Note**: Works system-wide
- **Note**: Also available via System Settings slider
