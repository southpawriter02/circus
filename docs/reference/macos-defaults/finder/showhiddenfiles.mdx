# Finder Show Hidden Files

Display hidden files and folders in Finder.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.finder` |
| **Key** | `AppleShowAllFiles` |
| **Type** | Boolean |
| **Default Value** | `false` |
| **Our Setting** | `true` |

## Description

Controls whether Finder displays hidden files and folders (those starting with a dot, like `.gitignore`, `.zshrc`, `.ssh`). Hidden by default to protect casual users from accidentally modifying system files, but essential for developers who work with dotfiles constantly.

## Values

| Value | Display |
|-------|---------|
| `false` | **Hide hidden files** (default) â€” Clean view |
| `true` | **Show hidden files** â€” All files visible |

## Our Configuration

**Set to**: `true` â€” Show hidden files.

### Why Show Hidden Files?

1. **Developer essential** â€” Access dotfiles
2. **Git** â€” See `.git`, `.gitignore`
3. **Config files** â€” Edit `.zshrc`, `.vimrc`
4. **Debugging** â€” Find hidden caches
5. **Full picture** â€” Know what's really there

### Why Hide Hidden Files?

- Cleaner appearance
- Prevent accidental modification
- Non-technical users
- Focused file browsing

## Common Hidden Files

| File/Folder | Purpose |
|-------------|---------|
| `.git` | Git repository data |
| `.gitignore` | Git ignore patterns |
| `.zshrc` | Zsh configuration |
| `.bash_profile` | Bash configuration |
| `.ssh` | SSH keys and config |
| `.config` | App configurations |
| `.npm` | npm cache |
| `.vscode` | VS Code workspace settings |
| `Library` | User library (partially hidden) |

## Code Examples

```bash
# Read current value
defaults read com.apple.finder AppleShowAllFiles

# Show hidden files (recommended for developers)
defaults write com.apple.finder AppleShowAllFiles -bool true

# Hide hidden files (default)
defaults write com.apple.finder AppleShowAllFiles -bool false

# Restart Finder to apply
killall Finder
```

## Terminal Alternative

View hidden files in Terminal regardless of Finder setting:
```bash
# List all files including hidden
ls -la

# List only hidden files
ls -la .*
```

## Security Considerations

When hidden files visible:
- `.ssh` folder exposed (contains keys)
- Config files with potential secrets visible
- Be careful in screen shares/screenshots

### Recommendation
Show hidden files for work, but be aware during presentations.

## The Library Folder

`~/Library` is special:
- Not a dotfile (not `.Library`)
- Hidden by different mechanism
- Access via Go â†’ Go to Folder â†’ `~/Library`

This setting may not reveal Library (depends on macOS version).
