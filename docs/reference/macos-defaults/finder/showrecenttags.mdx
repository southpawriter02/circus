# Finder Show Recent Tags

Display recently used tags in the Finder sidebar.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.finder` |
| **Key** | `ShowRecentTags` |
| **Type** | Boolean |
| **Default Value** | `true` |
| **Our Setting** | `true` |

## Description

Controls whether Finder displays a "Recent Tags" section in the sidebar, showing tags you've recently applied to files. This provides quick access to recently used organizational tags without navigating to the Tags section. Tags help organize files across different folders by applying colored labels and custom text tags.

## Values

| Value | Behavior |
|-------|----------|
| `true` | **Show recent tags** (default) â€” Recent tags in sidebar |
| `false` | **Hide recent tags** â€” Cleaner sidebar |

## Our Configuration

**Set to**: `true` â€” Show recent tags for quick organization.

### Why Show Recent Tags?

1. **Quick access** â€” Recently used tags at fingertips
2. **Workflow** â€” Continue organizing with same tags
3. **Discovery** â€” Remember what tags you've used
4. **Efficiency** â€” Fewer clicks to apply common tags
5. **Organization** â€” Encourages tag usage

### Why Hide Recent Tags?

- Sidebar space is limited
- Don't use tags frequently
- Prefer cleaner interface
- Tags section is sufficient
- Use search for tags instead

## Code Examples

```bash
# Read current value
defaults read com.apple.finder ShowRecentTags

# Show recent tags (default)
defaults write com.apple.finder ShowRecentTags -bool true

# Hide recent tags
defaults write com.apple.finder ShowRecentTags -bool false

# Delete (reset to default)
defaults delete com.apple.finder ShowRecentTags

# Restart Finder to apply
killall Finder
```
