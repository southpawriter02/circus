# ShowPathbar

Show the full file path at the bottom of Finder windows.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.finder` |
| **Key** | `ShowPathbar` |
| **Type** | Boolean |
| **Default Value** | `false` |
| **Our Setting** | `true` |

## Description

Displays a clickable path bar at the bottom of Finder windows showing the full hierarchy from root to current folder. Each component is clickable for quick navigation.

## Values

| Value | Behavior |
|-------|----------|
| `true` | **Show path bar** — Full path visible at window bottom |
| `false` | **Hide path bar** (default) — No path displayed |

## Our Configuration

**Set to**: `true` — Essential for navigation awareness.

## Code Examples

```bash
# Read current value
defaults read com.apple.finder ShowPathbar

# Show path bar
defaults write com.apple.finder ShowPathbar -bool true

# Hide path bar
defaults write com.apple.finder ShowPathbar -bool false

# Delete (reset to default)
defaults delete com.apple.finder ShowPathbar

# Restart Finder to apply
killall Finder
```

## UI Location

**View** menu → **Show Path Bar** (⌥⌘P)

## Using the Path Bar

- **Click** any folder to navigate there
- **Double-click** to open in new window
- **Command-click** for full disk path
- **Right-click** for context menu

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Path bar not showing | Restart Finder: `killall Finder` |
| Path bar too small | Cannot resize; it auto-sizes |
| Path not clickable | Ensure click is on folder name |

## Compatibility

- **Finder**: macOS 10.5+
- **Applies to**: All Finder windows

## Source File

**File**: [`defaults/interface/finder.sh`](../../../defaults/interface/finder.sh)
