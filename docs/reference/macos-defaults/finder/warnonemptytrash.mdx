# Finder Warn Before Emptying Trash

Control whether Finder shows a confirmation dialog before emptying the Trash.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.finder` |
| **Key** | `WarnOnEmptyTrash` |
| **Type** | Boolean |
| **Default Value** | `true` |
| **Our Setting** | `true` |

## Description

Controls whether Finder displays a confirmation dialog when you attempt to empty the Trash. When enabled, you'll see a warning showing how many items will be permanently deleted, giving you a chance to cancel if you've accidentally moved something important to Trash. Since emptying Trash is irreversible (without third-party recovery tools), this warning serves as a last line of defense against accidental data loss.

## Values

| Value | Behavior |
|-------|----------|
| `true` | **Show warning** (default) â€” Confirmation dialog appears |
| `false` | **No warning** â€” Trash empties immediately |

## Our Configuration

**Set to**: `true` â€” Show warning for safety.

### Why Keep Warning Enabled?

1. **Prevent data loss** â€” Last chance to save files
2. **Review before delete** â€” See item count and size
3. **Easy mistakes** â€” Keyboard shortcut is close to Move to Trash
4. **No undo** â€” Empty Trash cannot be reversed
5. **Small inconvenience** â€” One click for peace of mind

### Why Disable Warning?

- **Power users** â€” Confident in Trash habits
- **Workflow efficiency** â€” Regularly empty Trash
- **Already have backups** â€” Time Machine running
- **Quick cleanup** â€” Frequently clearing known junk
- **Annoyance** â€” Find the dialog disruptive

## Code Examples

```bash
# Read current value
defaults read com.apple.finder WarnOnEmptyTrash

# Enable warning (default, recommended)
defaults write com.apple.finder WarnOnEmptyTrash -bool true

# Disable warning (use with caution)
defaults write com.apple.finder WarnOnEmptyTrash -bool false

# Delete (reset to default)
defaults delete com.apple.finder WarnOnEmptyTrash

# Restart Finder to apply
killall Finder
```

## Related Settings

```bash
# Warn before emptying Trash (this setting)
defaults write com.apple.finder WarnOnEmptyTrash -bool true

# Auto-empty Trash after 30 days
defaults write com.apple.finder FXRemoveOldTrashItems -bool false

# Show warning before changing extension
defaults write com.apple.finder FXEnableExtensionChangeWarning -bool true

# Show warning before removing from iCloud Drive
defaults write com.apple.finder FXEnableRemoveFromICloudDriveWarning -bool true

# Secure empty trash (pre-macOS 10.11, now deprecated)
# defaults write com.apple.finder EmptyTrashSecurely -bool true
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Warning not appearing | Check Finder > Settings > Advanced |
| Want to disable temporarily | Hold Option key while selecting Empty Trash |
| Trash won't empty | Some files may be locked or in use |
| Recovered files not in Trash | Time Machine or third-party recovery needed |
| Setting not persisting | Run `killall Finder` after changing |
| "Trash cannot be emptied" | Check for locked files or active file access |

## Compatibility

- **macOS**: 10.0+ (all macOS versions)
- **Note**: Option key bypasses warning temporarily
- **Note**: "Secure Empty Trash" removed in macOS 10.11+
- **Note**: Auto-empty after 30 days is separate setting
- **Note**: Restart Finder for changes to apply
