# _FXShowPosixPathInTitle

Display the full POSIX path in Finder window title bars.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.finder` |
| **Key** | `_FXShowPosixPathInTitle` |
| **Type** | Boolean |
| **Default Value** | `false` |
| **Our Setting** | `true` |
| **Requires Restart** | Finder restart |

## Description

Controls whether Finder window title bars display the full POSIX path (e.g., `/Users/username/Documents/Project`) instead of just the folder name (e.g., `Project`). This is an undocumented but stable preference that power users rely on for efficient navigation.

When enabled, you can instantly see the complete path to your current location, making it easy to:

- **Understand context**: Know exactly where you are in the filesystem
- **Copy paths**: âŒ˜-click the title to reveal path components, or copy the full path
- **Navigate**: Click path components in the title to jump to parent directories
- **Debug**: Verify you're looking at the correct location, especially with similarly-named folders
- **Terminal integration**: Quickly see paths for use in command-line operations

## Values

| Value | Behavior |
|-------|----------|
| `true` | **Full path shown** â€” Title displays `/Users/username/Documents/Folder` |
| `false` | **Folder name only** (default) â€” Title displays just `Folder` |

## Default Behavior

**Apple's default**: `false` (folder name only)

Apple's design philosophy favors clean, minimal UI. The folder-name-only approach looks tidier but sacrifices context, especially when working with multiple windows or similarly-named folders.

## Our Configuration

**Set to**: `true`

**Rationale**: Developers and power users frequently work with multiple windows open to different locations. Seeing the full path eliminates confusion and provides instant context. The extra information is worth the slightly longer title bar text.

This is particularly valuable when:
- Working on multiple projects with similar folder structures
- Navigating deep directory hierarchies
- Copying files between locations
- Debugging "where did I put that?" situations

## Code Examples

### Read Current Value

```bash
# Check current setting
defaults read com.apple.finder _FXShowPosixPathInTitle

# Returns: 1 (enabled) or 0 (disabled)
# Note: May error if never set; absence means false
```

### Enable Full Path in Title

```bash
# Show full POSIX path in title bar
defaults write com.apple.finder _FXShowPosixPathInTitle -bool true

# Restart Finder to apply
killall Finder
```

### Disable Full Path in Title

```bash
# Show folder name only (Apple's default)
defaults write com.apple.finder _FXShowPosixPathInTitle -bool false

# Restart Finder to apply
killall Finder
```

### Delete (Reset to System Default)

```bash
# Remove custom setting, restore to system default
defaults delete com.apple.finder _FXShowPosixPathInTitle

# Restart Finder to apply
killall Finder
```

## Behavior Details

### Path Format

The path shown is the POSIX (Unix-style) path:
- Starts with `/`
- Uses forward slashes as separators
- Shows the actual filesystem path, not aliases

### Home Directory

Your home directory is shown as `/Users/username`, not `~`:

```
/Users/alex/Documents/Project
```

### Network and External Volumes

For mounted volumes, the path reflects the mount point:

```
/Volumes/External Drive/Projects/MyProject
/Volumes/server.local/shared/documents
```

### Long Paths

Very long paths may be truncated in narrow windows. The window title shows as much as space allows, with the rightmost (current folder) portion prioritized.

## Troubleshooting

### Path Not Showing After Enabling

1. Verify the setting:
   ```bash
   defaults read com.apple.finder _FXShowPosixPathInTitle
   # Should return 1
   ```

2. Restart Finder:
   ```bash
   killall Finder
   ```

3. Check that you're viewing a folder window, not search results or a special view

### Path Is Truncated

- Widen the Finder window
- The path prioritizes showing the current location; parent directories may truncate
- Consider using Path Bar for always-visible full path

### Setting Reverts After Restart

1. Check for MDM profiles that might override:
   ```bash
   sudo profiles list
   ```

2. Verify you're writing to user preferences:
   ```bash
   defaults read com.apple.finder | grep _FXShowPosixPathInTitle
   ```

## Developer Notes

### Preference Key

The underscore prefix (`_FX`) indicates this is an internal/undocumented preference. Despite this, it has remained stable across macOS versions and is widely used.

### Detecting in Code
