# QLEnableTextSelection

Enable text selection in Quick Look previews.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.finder` |
| **Key** | `QLEnableTextSelection` |
| **Type** | Boolean |
| **Default Value** | `false` |
| **Our Setting** | `true` |

## Description

Enables selecting and copying text from Quick Look previews (spacebar preview). By default, Quick Look only displays text but doesn't allow selection.

## Values

| Value | Behavior |
|-------|----------|
| `true` | **Text selectable** — Can copy text from previews |
| `false` | **View only** (default) — No text selection allowed |

## Our Configuration

**Set to**: `true` — Copy text directly from Quick Look without opening files.

## Code Examples

```bash
# Read current value
defaults read com.apple.finder QLEnableTextSelection

# Enable text selection in Quick Look
defaults write com.apple.finder QLEnableTextSelection -bool true

# Disable (Apple default)
defaults write com.apple.finder QLEnableTextSelection -bool false

# Delete (reset to default)
defaults delete com.apple.finder QLEnableTextSelection

# Restart Finder to apply
killall Finder
```

## UI Location

Not available in UI — terminal only.

## How to Use

1. Select a file in Finder
2. Press **Space** to open Quick Look
3. Select text with mouse
4. Press ⌘C to copy

## Supported File Types

Text selection works in:
- Plain text files (.txt, .md, .json, etc.)
- PDFs
- RTF documents
- Some code files

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Text not selectable | Restart Finder: `killall Finder` |
| Only works in some files | Not all file types support selection |
| Selection cursor missing | File type may not support text selection |

## Compatibility

- **Finder**: macOS 10.5+
- **Note**: Not all file types support text selection
