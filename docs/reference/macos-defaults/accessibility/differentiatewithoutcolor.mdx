# Differentiate Without Color

Add shapes and labels to elements that use color alone to convey information.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.universalaccess` |
| **Key** | `differentiateWithoutColor` |
| **Type** | Boolean |
| **Default Value** | `false` |
| **Our Setting** | `false` |

## Description

When enabled, macOS adds shapes, icons, or other visual indicators to interface elements that otherwise rely solely on color to convey meaning. This ensures that users with color vision deficiencies (color blindness) can still understand the status or meaning of colored elements. For example, a red error indicator might also show an X icon, or a green success indicator might include a checkmark.

## Values

| Value | Behavior |
|-------|----------|
| `true` | **Add shapes/labels** â€” Color + shape indicators |
| `false` | **Color only** (default) â€” Standard appearance |

## Our Configuration

**Set to**: `false` â€” Standard color-based indicators.

### Why Enable Differentiate Without Color?

1. **Color blindness** â€” Essential for color vision deficiency
2. **Accessibility** â€” WCAG compliance support
3. **Clarity** â€” Redundant cues aid understanding
4. **Grayscale** â€” Works with grayscale displays
5. **Low light** â€” Colors less distinguishable in dim lighting

### Why Disable?

- Normal color vision
- Prefer cleaner interface
- Shapes add visual noise
- Testing purposes only
- Third-party app inconsistency

## Code Examples

```bash
# Read current value
defaults read com.apple.universalaccess differentiateWithoutColor

# Enable differentiate without color
defaults write com.apple.universalaccess differentiateWithoutColor -bool true

# Disable (default)
defaults write com.apple.universalaccess differentiateWithoutColor -bool false

# Delete (reset to default)
defaults delete com.apple.universalaccess differentiateWithoutColor
```

## App Support

This setting primarily affects:
- **Apple Apps**: Calendar, Finder, Mail, Messages, etc.
- **System UI**: Alerts, notifications, status indicators
- **Some third-party apps**: That read accessibility settings

Apps that may not support:
- Web content (websites control their own colors)
- Games and media apps
- Legacy applications
- Apps that don't use standard UI controls

## Developer Guidance

For app developers:

```swift
// Check if user has enabled this setting
if UIAccessibility.shouldDifferentiateWithoutColor {
    // Add shapes/icons alongside color indicators
    indicator.addSymbol(.checkmark)
}
```

Design best practices:
- Never rely on color alone
- Use icons + color + text when possible
- Test with color filters enabled
- Follow WCAG 2.1 guidelines

## Combined With Color Filters

For maximum accessibility:

```bash
# Enable differentiate without color
defaults write com.apple.universalaccess differentiateWithoutColor -bool true

# PLUS add color filter for your type of color blindness
defaults write com.apple.universalaccess colorFilterEnabled -bool true

# Type 2 = Deuteranopia (most common)
defaults write com.apple.universalaccess colorFilterType -int 2
```
