# AppleShowScrollBars

Control when scrollbars are visible.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `NSGlobalDomain` |
| **Key** | `AppleShowScrollBars` |
| **Type** | String |
| **Default Value** | `Automatic` |
| **Our Setting** | `Always` |

## Description

Controls when scrollbars appear in windows and scrollable views across all applications.

## Values

| Value | Behavior |
|-------|----------|
| `Always` | **Always visible** — Scrollbars shown at all times |
| `Automatic` | **Auto** (default) — Based on input device (hidden with trackpad) |
| `WhenScrolling` | **When scrolling** — Only visible during active scrolling |

## Our Configuration

**Set to**: `Always` — Constant awareness of document position, especially useful with mice.

## Code Examples

```bash
# Read current value
defaults read NSGlobalDomain AppleShowScrollBars

# Always show scrollbars
defaults write NSGlobalDomain AppleShowScrollBars -string "Always"

# Show based on input device (Apple default)
defaults write NSGlobalDomain AppleShowScrollBars -string "Automatic"

# Show only when scrolling
defaults write NSGlobalDomain AppleShowScrollBars -string "WhenScrolling"

# Delete (reset to default)
defaults delete NSGlobalDomain AppleShowScrollBars
```

## UI Location

**System Settings** → **Appearance** → **Show scroll bars**

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Scrollbars still hidden | Restart apps to apply change |
| Scrollbars thick/thin | Controlled by system appearance |
| Per-app behavior differs | Some apps override system settings |

## Compatibility

- **macOS**: 10.7+
- **Affects**: All Cocoa applications
