# Type to Siri

Enable typing queries to Siri instead of speaking.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.Siri` |
| **Key** | `TypeToSiriEnabled` |
| **Type** | Boolean |
| **Default Value** | `false` |
| **Our Setting** | `true` |

## Description

Controls whether you can type queries to Siri instead of speaking them. When enabled, activating Siri (via keyboard shortcut or menu bar) shows a text input field instead of starting voice recognition. This is useful in quiet environments, public spaces, or for users who prefer typing.

## Values

| Value | Behavior |
|-------|----------|
| `false` | **Voice only** (default) — Siri listens for spoken queries |
| `true` | **Type to Siri** — Text input field appears |

## Our Configuration

**Set to**: `true` — Enable Type to Siri for discreet queries.

### Why Enable Type to Siri?

1. **Privacy** — No spoken queries overheard
2. **Quiet environments** — Libraries, offices, bedrooms
3. **Accuracy** — Avoid voice recognition errors
4. **Complex queries** — Easier to type long requests
5. **Discretion** — Private searches without speaking aloud

### Why Keep Voice Mode?

- Hands-free operation needed
- Quick commands preferred
- Accessibility requirements
- HomeKit voice control
- Driving or cooking

## Code Examples

```bash
# Read current value
defaults read com.apple.Siri TypeToSiriEnabled

# Enable Type to Siri
defaults write com.apple.Siri TypeToSiriEnabled -bool true

# Disable (voice mode)
defaults write com.apple.Siri TypeToSiriEnabled -bool false

# Delete (reset to default)
defaults delete com.apple.Siri TypeToSiriEnabled
```

Changes take effect immediately for new Siri activations.

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Still shows voice mode | Restart Siri: `killall Siri` |
| Setting doesn't persist | Check if Siri is enabled first |
| Text field not appearing | Verify setting with `defaults read` |
| Can't find in System Settings | Located under Accessibility > Siri |
| Want both modes | Not possible - choose one default |

## Voice + Type Hybrid

While you can't have both simultaneously, you can:

1. Enable Type to Siri as default
2. Use "Hey Siri" for voice (if enabled)
3. "Hey Siri" always uses voice regardless of this setting

```
+-------------------------------------------------------------+
|  Activation Methods with Type to Siri Enabled               |
+-------------------------------------------------------------+
|                                                              |
|  Keyboard shortcut  -->  Type interface                     |
|  Menu bar click     -->  Type interface                     |
|  "Hey Siri"         -->  Voice interface (always)           |
|                                                              |
|  This allows flexibility: type when convenient,             |
|  voice when hands-free is needed                            |
|                                                              |
+-------------------------------------------------------------+
```

## Compatibility

- **macOS**: 10.14+ (Mojave)
- **Note**: Requires Siri to be enabled
- **Note**: Takes effect immediately
- **Note**: "Hey Siri" bypasses this setting
