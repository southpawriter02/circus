# NSNavPanelExpandedStateForSaveMode

Expand the Save dialog by default.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `NSGlobalDomain` |
| **Key** | `NSNavPanelExpandedStateForSaveMode` / `NSNavPanelExpandedStateForSaveMode2` |
| **Type** | Boolean |
| **Default Value** | `false` |
| **Our Setting** | `true` |

## Description

Controls whether the Save dialog shows the expanded view with full file browser, or the condensed view with just a filename field.

## Values

| Value | Behavior |
|-------|----------|
| `true` | **Expanded** — Full file browser with folder navigation |
| `false` | **Condensed** (default) — Minimal view, just filename |

## Our Configuration

**Set to**: `true` — Full navigator for easier file management.

## Code Examples

```bash
# Read current value
defaults read NSGlobalDomain NSNavPanelExpandedStateForSaveMode

# Expand Save dialogs by default (set both for compatibility)
defaults write NSGlobalDomain NSNavPanelExpandedStateForSaveMode -bool true
defaults write NSGlobalDomain NSNavPanelExpandedStateForSaveMode2 -bool true

# Delete (reset to default)
defaults delete NSGlobalDomain NSNavPanelExpandedStateForSaveMode
defaults delete NSGlobalDomain NSNavPanelExpandedStateForSaveMode2
```

## UI Location

When saving a file (⌘S), click the disclosure arrow to toggle. This setting changes the default state.

## Related: Expand Print Dialog

```bash
# Also expand Print dialogs
defaults write NSGlobalDomain PMPrintingExpandedStateForPrint -bool true
defaults write NSGlobalDomain PMPrintingExpandedStateForPrint2 -bool true
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Dialog still condensed | Set both keys (with and without "2") |
| Per-app behavior differs | Some apps override system setting |
| Setting resets | Log out and back in |

## Compatibility

- **Dialogs**: macOS 10.6+
- **Applies to**: All native Save dialogs
