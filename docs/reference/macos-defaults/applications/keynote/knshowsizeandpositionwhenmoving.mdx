# Keynote Show Size and Position When Moving

Control whether a tooltip displays object dimensions and coordinates while dragging objects in Keynote.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.iWork.Keynote` |
| **Key** | `KNShowSizeAndPositionWhenMoving` |
| **Type** | Boolean |
| **Default Value** | `true` |
| **Our Setting** | `true` |

## Description

Controls whether Keynote displays a real-time tooltip showing an object's size (width x height) and position (X, Y coordinates) while you drag it on the slide canvas. This information helps you precisely position and size objects without constantly checking the Format inspector.

## Values

| Value | Behavior |
|-------|----------|
| `true` | **Show info** (default) — Tooltip displays during drag |
| `false` | **Hide info** — No tooltip while moving objects |

## Our Configuration

**Set to**: `true` — Show real-time position and size information.

### Why Enable This?

1. **Precise positioning** — Know exact coordinates while moving
2. **Consistent sizing** — Match object dimensions easily
3. **Faster workflow** — No need to check Format inspector
4. **Grid-free alignment** — Position without visible grid
5. **Professional layouts** — Ensure exact measurements

### Why Disable This?

- **Cleaner canvas** — Less visual clutter during editing
- **Approximate positioning** — Exact numbers not important
- **Smaller displays** — Tooltip may obstruct view

## Code Examples

```bash
# Read current value
defaults read com.apple.iWork.Keynote KNShowSizeAndPositionWhenMoving

# Enable position/size tooltip (default, recommended)
defaults write com.apple.iWork.Keynote KNShowSizeAndPositionWhenMoving -bool true

# Disable position/size tooltip
defaults write com.apple.iWork.Keynote KNShowSizeAndPositionWhenMoving -bool false

# Delete (reset to default)
defaults delete com.apple.iWork.Keynote KNShowSizeAndPositionWhenMoving
```

## Related Settings

```bash
defaults write com.apple.iWork.Keynote KNShowSizeAndPositionWhenMoving -bool true

# Object center alignment guides
defaults write com.apple.iWork.Keynote KNShowsObjectCenterGuides -bool true

# Object edge alignment guides
defaults write com.apple.iWork.Keynote KNShowsObjectEdgeGuides -bool true

# Show rulers
defaults write com.apple.iWork.Keynote TSShowsRuler -bool true

# Default zoom level
defaults write com.apple.iWork.Keynote TSDefaultZoom -int 100
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Tooltip not appearing | Check Settings > Rulers > Show size and position |
| Units wrong (inches vs points) | Change Ruler Units in Settings > Rulers |
| Tooltip blocks view | Temporarily disable or move object differently |
| Want exact position | Use Format sidebar > Arrange for precise input |
| Tooltip shows wrong position | Ensure you're dragging the object, not a handle |
| Setting not persisting | Restart Keynote after changing via defaults |

## Compatibility

- **macOS**: 10.14+ (Mojave) with Keynote
- **Note**: Similar setting exists in Pages and Numbers
- **Note**: Restart Keynote for terminal changes to apply
- **Note**: Tooltip only appears during active drag operations
- **Note**: Units match the Ruler Units setting (points, inches, cm)
