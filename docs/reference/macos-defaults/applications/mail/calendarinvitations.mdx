# Mail Calendar Invitations

Automatically add meeting invitations to Calendar.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.mail` |
| **Key** | `AddInvitationsToCalendar` |
| **Type** | Boolean |
| **Default Value** | `true` |
| **Our Setting** | `true` |

## Description

Controls whether Mail automatically adds calendar invitations (ICS files) received via email to the Calendar app. When enabled, meeting invitations appear in Calendar immediately, showing as tentative until you respond. This provides seamless integration between email and calendar without manual importing.

## Values

| Value | Behavior |
|-------|----------|
| `true` | **Auto-add invitations** (default) â€” Events appear in Calendar |
| `false` | **Manual handling** â€” Must manually add to calendar |

## Our Configuration

**Set to**: `true` â€” Automatically add invitations for convenience.

### Why Enable Auto-Add?

1. **Convenience** â€” Events appear automatically
2. **Visibility** â€” See conflicts immediately
3. **No manual steps** â€” Seamless workflow
4. **Time-saving** â€” One less thing to manage
5. **Standard behavior** â€” Expected in modern email

### Why Disable Auto-Add?

- Want to review before adding
- Multiple calendar apps (conflicts)
- Spam calendar invitations
- Manual calendar management preference
- Different calendar system

## Security Implications

| Risk | Impact | Mitigation |
|------|--------|------------|
| **Calendar spam** | Unwanted events in calendar | Events from unknown senders still require accept |
| **Phishing** | Fake meeting invites | Review sender before accepting |
| **Clutter** | Many tentative events | Decline promptly |

### Best Practices

- Review sender before accepting
- Decline unwanted invitations promptly
- Check meeting links carefully
- Report calendar spam

## Code Examples

```bash
# Read current value
defaults read com.apple.mail AddInvitationsToCalendar

# Enable auto-add (default, recommended)
defaults write com.apple.mail AddInvitationsToCalendar -bool true

# Disable auto-add
defaults write com.apple.mail AddInvitationsToCalendar -bool false

# Delete (reset to default)
defaults delete com.apple.mail AddInvitationsToCalendar
```

## Response Options

When viewing an invitation in Mail:
- **Accept** â€” Event confirmed, response sent
- **Maybe/Tentative** â€” Event stays tentative
- **Decline** â€” Event removed from calendar

Response is sent back to organizer via email.

## ICS File Format

Calendar invitations use iCalendar format (.ics):
```
BEGIN:VCALENDAR
VERSION:2.0
BEGIN:VEVENT
DTSTART:20260114T140000
DTEND:20260114T150000
SUMMARY:Team Meeting
LOCATION:Conference Room A
ORGANIZER:mailto:boss@company.com
ATTENDEE:mailto:you@company.com
END:VEVENT
END:VCALENDAR
```

## Multiple Calendar Accounts

When you have multiple calendars:
1. Invitation added to **default** calendar
2. Can move to different calendar after accepting
3. Response sent from email account that received invite

## Exchange/Office 365 Integration

For Exchange accounts:
- Invitations handled by Exchange protocol
- May bypass this Mail setting
- Check Exchange account settings separately

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Invitations not appearing | Verify setting is enabled |
| Wrong calendar used | Change default calendar |
| Can't respond to invite | Check email account permissions |
| Duplicate events | Check multiple email clients |
| Exchange invites different | Exchange has own handling |
