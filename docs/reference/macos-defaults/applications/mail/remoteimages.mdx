# Mail Block Remote Images

Block loading of remote images in emails for privacy.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.mail` |
| **Key** | `DisableURLLoading` |
| **Type** | Boolean |
| **Default Value** | `false` |
| **Our Setting** | `true` |

## Description

Controls whether Mail automatically loads remote images and content embedded in emails. Remote images are often used by marketers and malicious actors to track when and where emails are opened. Blocking remote content prevents this tracking and protects your privacy, while also saving bandwidth and speeding up email loading.

## Values

| Value | Behavior |
|-------|----------|
| `true` | **Block remote images** â€” Images must be manually loaded |
| `false` | **Allow remote images** (default) â€” Auto-load all content |

## Our Configuration

**Set to**: `true` â€” Block remote images by default.

### Why Block Remote Images?

1. **Privacy** â€” Prevents email open tracking
2. **Location hiding** â€” Your IP address stays private
3. **Speed** â€” Emails load faster
4. **Bandwidth** â€” Saves data on metered connections
5. **Security** â€” Blocks potential malicious content
6. **Phishing protection** â€” Reveals suspicious emails

### Why Allow Remote Images?

- Better email formatting
- See product images immediately
- Newsletters display correctly
- Less manual interaction needed

## Security Implications

| Threat | Protection |
|--------|------------|
| **Tracking pixels** | Blocked â€” sender can't confirm open |
| **IP address exposure** | Protected â€” no connection to sender's server |
| **Location tracking** | Prevented â€” IP geolocation blocked |
| **Malicious images** | Blocked â€” can't exploit image parsers |
| **Web beacons** | Blocked â€” marketing analytics defeated |

## Code Examples

```bash
# Read current value
defaults read com.apple.mail DisableURLLoading

# Block remote images (recommended for privacy)
defaults write com.apple.mail DisableURLLoading -bool true

# Allow remote images (default)
defaults write com.apple.mail DisableURLLoading -bool false

# Delete (reset to default)
defaults delete com.apple.mail DisableURLLoading
```

## Mail Privacy Protection vs Block Remote Content

| Feature | Block Remote Content | Mail Privacy Protection |
|---------|---------------------|------------------------|
| **How it works** | Blocks all remote loads | Routes through Apple proxy |
| **Image display** | Must manually load | Images load normally |
| **IP hidden** | Yes (nothing loads) | Yes (Apple proxy) |
| **Open tracking** | Blocked | Pre-fetches to mask opens |
| **Convenience** | Lower | Higher |

## Per-Sender Exceptions

Even with blocking enabled, you can load remote content:
1. Click "Load Remote Content" banner in email
2. This is per-email, not remembered

For trusted senders, consider Mail Privacy Protection instead.

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Can't see newsletter images | Click "Load Remote Content" or add sender exception |
| Emails look broken | Remote images blocked; load selectively |
| Setting not applying | Restart Mail app |
| Want images but privacy | Use Mail Privacy Protection instead |

| Email Type | Impact | Recommendation |

| Personal email | Low â€” mostly text | Minimal impact |
