# Mail Number of Snippet Lines

Control how many lines of message preview appear in the message list.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.mail` |
| **Key** | `NumberOfSnippetLines` |
| **Type** | Integer |
| **Default Value** | `2` |
| **Our Setting** | `2` |

## Description

Controls how many lines of the email body are shown as a preview snippet in the Mail message list. This preview appears below the sender and subject line, giving you a glimpse of the email content without opening it. More lines provide more context but show fewer messages on screen; fewer lines allow more messages visible at once but with less preview content.

## Values

| Value | Behavior |
|-------|----------|
| `0` | **No preview** — Only sender and subject shown |
| `1` | **One line** — Minimal preview, maximum messages |
| `2` | **Two lines** (default) — Balanced preview |
| `3` | **Three lines** — More context per message |
| `4` | **Four lines** — Detailed preview |
| `5` | **Five lines** — Maximum preview content |

## Our Configuration

**Set to**: `2` — Default balanced preview with good visibility.

### Why Use Default?

1. **Balance** — Enough context without crowding
2. **Efficient** — See multiple emails at once
3. **Quick triage** — Determine priority without opening
4. **Standard** — Expected Mail behavior

### Why Decrease?

- Process high email volume quickly
- Prefer to open emails for content
- Want maximum emails visible
- Smaller screen/display

### Why Increase?

- Low email volume
- Want to preview without opening
- Large display with space to spare
- Prefer detailed message list

## Code Examples

```bash
# Read current value
defaults read com.apple.mail NumberOfSnippetLines

# Set to no preview (0)
defaults write com.apple.mail NumberOfSnippetLines -int 0

# Set to default (2 lines)
defaults write com.apple.mail NumberOfSnippetLines -int 2

# Set to maximum (5 lines)
defaults write com.apple.mail NumberOfSnippetLines -int 5

# Delete (reset to default)
defaults delete com.apple.mail NumberOfSnippetLines
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Preview not showing | Verify setting is > 0; restart Mail |
| Preview shows attachments | Normal behavior; shows text content |
| HTML emails show code | Preview shows plain text version |
| Changes not applying | Quit and restart Mail app |
| Want different per-mailbox | Not supported; global setting |

## Compatibility

- **macOS**: 10.10+ (Yosemite) and later
- **Note**: Setting applies to all mailboxes
- **Note**: Restart Mail for changes to take effect
- **Note**: Preview shows plain text, not HTML formatting
- **Note**: Syncs with iOS Mail on some versions
