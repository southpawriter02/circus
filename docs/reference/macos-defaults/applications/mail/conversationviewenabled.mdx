# Mail Conversation View

Group related emails into threaded conversations.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.mail` |
| **Key** | `ConversationViewEnabled` |
| **Type** | Boolean |
| **Default Value** | `true` |
| **Our Setting** | `true` |

## Description

Controls whether Mail groups related messages into threaded conversations. When enabled, emails with the same subject line and participants are collapsed into a single conversation entry, showing the most recent message with a count of total messages in the thread. This makes it easier to follow email discussions and reduces inbox clutter from back-and-forth exchanges.

## Values

| Value | Behavior |
|-------|----------|
| `true` | **Conversation view** (default) â€” Group related emails into threads |
| `false` | **Individual messages** â€” Show each email as a separate item |

## Our Configuration

**Set to**: `true` â€” Enable conversation threading for organized inbox.

### Why Enable Conversations?

1. **Organization** â€” Related emails grouped together
2. **Context** â€” See full discussion history at once
3. **Cleaner inbox** â€” Fewer items to scroll through
4. **Follow-up** â€” Easy to track conversation status
5. **Standard** â€” Matches modern email behavior

### Why Disable Conversations?

- Prefer chronological email list
- Each email needs individual attention
- Conversations split unexpectedly
- Work with automated/system emails
- Old email workflow habit

## Code Examples

```bash
# Read current value
defaults read com.apple.mail ConversationViewEnabled

# Enable conversation view (recommended)
defaults write com.apple.mail ConversationViewEnabled -bool true

# Disable conversation view
defaults write com.apple.mail ConversationViewEnabled -bool false

# Delete (reset to default)
defaults delete com.apple.mail ConversationViewEnabled
```


| Issue | Solution |
|-------|----------|

| Setting not applying | Restart Mail app |
