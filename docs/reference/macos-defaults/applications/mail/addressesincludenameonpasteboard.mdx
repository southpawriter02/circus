# Mail Addresses Include Name on Pasteboard

Control whether copied email addresses include the contact name.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.mail` |
| **Key** | `AddressesIncludeNameOnPasteboard` |
| **Type** | Boolean |
| **Default Value** | `false` |
| **Our Setting** | `false` |

## Description

Controls whether copying an email address from Mail includes the contact's display name along with the email address. When enabled, copying "john@example.com" from a message would copy "John Smith <john@example.com>" instead of just the email address. This affects how email addresses can be pasted into other applications.

## Values

| Value | Behavior |
|-------|----------|
| `false` | **Email only** (default) — Copies just the email address |
| `true` | **Name and email** — Copies "Name <email@domain.com>" format |

## Our Configuration

**Set to**: `false` — Copy email address only for cleaner pasting.

### Why Email Only?

1. **Universal compatibility** — Works in all email fields
2. **Clean data** — No need to parse out the name
3. **Form friendly** — Valid for sign-up/contact forms
4. **Technical use** — Ready for scripts and configs
5. **Privacy** — Less personal information copied

### Why Include Name?

- Forwarding complete contact information
- Creating address book entries
- Professional email introductions
- When context about the person matters

## Code Examples

```bash
# Read current value
defaults read com.apple.mail AddressesIncludeNameOnPasteboard

# Copy email address only (default)
defaults write com.apple.mail AddressesIncludeNameOnPasteboard -bool false

# Copy name and email address
defaults write com.apple.mail AddressesIncludeNameOnPasteboard -bool true

# Delete (reset to default)
defaults delete com.apple.mail AddressesIncludeNameOnPasteboard
```

## UI Location

This setting is not directly exposed in Mail's preferences UI. It can only be configured via the `defaults` command or terminal.

```
+-------------------------------------------------------------+
|  Not Available in Mail Settings UI                           |
+-------------------------------------------------------------+
|                                                              |
|  This preference must be set via Terminal:                  |
|                                                              |
|  $ defaults write com.apple.mail \                          |
|      AddressesIncludeNameOnPasteboard -bool true            |
|                                                              |
|  Or using defaults read to check current value:             |
|                                                              |
|  $ defaults read com.apple.mail \                           |
|      AddressesIncludeNameOnPasteboard                       |
|                                                              |
|  Note: Restart Mail after changing for it to take effect    |
|                                                              |
+-------------------------------------------------------------+
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Setting not working | Restart Mail after changing via defaults |
| Still getting name | Use manual text selection instead of "Copy Address" |
| Pasted address invalid | Use false setting for cleaner email-only copy |
| Want both options | Toggle setting as needed, or use text selection |
| Can't find in UI | This is a hidden preference, use terminal only |

## Compatibility

- **macOS**: 10.10+ (Yosemite and later)
- **Note**: Hidden preference, not in Mail Settings UI
- **Note**: Affects "Copy Address" menu command
- **Note**: Restart Mail after changing
- **Note**: Manual text selection ignores this setting
