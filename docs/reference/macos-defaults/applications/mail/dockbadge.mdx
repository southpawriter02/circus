# Mail Dock Unread Badge

Display unread message count on the Mail Dock icon.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.mail` |
| **Key** | `ShouldShowUnreadMessagesInDockIcon` |
| **Type** | Boolean |
| **Default Value** | `true` |
| **Our Setting** | `true` |

## Description

Controls whether Mail displays a red badge on its Dock icon showing the number of unread messages. This provides an at-a-glance count of unread emails without opening the app. While useful for staying aware of new messages, it can also be a source of distraction and notification anxiety for some users.

## Values

| Value | Behavior |
|-------|----------|
| `true` | **Show badge** (default) â€” Unread count visible on Dock |
| `false` | **Hide badge** â€” No count shown, cleaner Dock |

## Our Configuration

**Set to**: `true` â€” Show unread badge for awareness.

### Why Show Dock Badge?

1. **Awareness** â€” Know when new mail arrives
2. **Quick glance** â€” Check count without opening app
3. **Priority indicator** â€” High count suggests attention needed
4. **Standard behavior** â€” Expected Mail app behavior
5. **No app switching** â€” See count while in other apps

### Why Hide Dock Badge?

- Reduce notification anxiety
- Minimize distractions during focus time
- Cleaner Dock appearance
- Already use notifications instead
- Process email on schedule, not reactively

## Code Examples

```bash
# Read current value
defaults read com.apple.mail ShouldShowUnreadMessagesInDockIcon

# Show unread badge (default)
defaults write com.apple.mail ShouldShowUnreadMessagesInDockIcon -bool true

# Hide unread badge
defaults write com.apple.mail ShouldShowUnreadMessagesInDockIcon -bool false

# Delete (reset to default)
defaults delete com.apple.mail ShouldShowUnreadMessagesInDockIcon
```

## What Counts as "Unread"

The badge counts:
- New, never-opened emails
- Emails manually marked as unread
- Emails in all mailboxes (Inbox, folders, etc.)

Not counted:
- Emails in Junk/Spam
- Emails in Trash
- Drafts

## Related Notification Settings


## Alternative: Use Focus Modes

Instead of permanently hiding the badge:

1. **Create a Focus Mode** (Work, Personal Time, etc.)
2. **Customize notifications** for that Focus
3. **Hide badge during Focus** but show normally otherwise

This gives you control based on context.

## Badge vs Other Notifications

| Notification Type | Intrusiveness | This Setting |
|-------------------|---------------|--------------|
| **Dock badge** | Low (passive) | Controls this |
| **Sound** | Medium | Separate setting |
| **Banner** | Medium-High | System Settings |
| **Alert** | High | System Settings |
| **Bounce icon** | Medium | Separate setting |

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Badge not updating | Restart Mail app |
| Badge stuck on old count | Rebuild mailbox |
| Want badge for one account only | Not possible - all or nothing |
| Badge shows wrong count | Check all mailboxes for unread |

## Count Not Matching Reality?

If badge count seems wrong:
1. Check **all mailboxes** for unread messages
2. Check **All Mail** or **Archive** folders
3. Rebuild mailbox: **Mailbox** â†’ **Rebuild**
4. Check email account sync status
