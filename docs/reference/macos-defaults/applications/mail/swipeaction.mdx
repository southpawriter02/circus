# Mail Swipe Action

Configure the action performed when swiping on messages in Mail.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.mail` |
| **Key** | `SwipeAction` |
| **Type** | Integer |
| **Default Value** | `1` |
| **Our Setting** | `1` |

## Description

Controls what action is performed when you swipe left or right on a message in the Mail message list using a trackpad or Magic Mouse. Swipe gestures provide quick message management without using menus or keyboard shortcuts. The setting determines whether swiping archives, deletes, or marks messages as read/unread.

## Values

| Value | Behavior |
|-------|----------|
| `0` | **None** â€” Disable swipe actions |
| `1` | **Archive/Delete** (default) â€” Swipe to archive (or delete if no archive) |
| `2` | **Mark Read/Unread** â€” Swipe to toggle read status |
| `3` | **Flag/Unflag** â€” Swipe to toggle flag status |

## Our Configuration

**Set to**: `1` â€” Archive/Delete for efficient inbox management.

### Why Use Archive/Delete?

1. **Efficiency** â€” Quick inbox zero workflow
2. **Common action** â€” Most frequent email operation
3. **Intuitive** â€” Matches iOS Mail behavior
4. **Reversible** â€” Archived messages easily retrieved

### Why Use Mark Read/Unread?

- Don't want accidental deletions
- Focus on reading through emails
- Archive manually when ready
- Track what needs attention

### Why Use Flag?

- Use flags for follow-up workflow
- Don't want to delete from list view
- Flag-based task management

### Why Disable?

- Prevent accidental actions
- Prefer keyboard shortcuts
- Find swipes unreliable
- Use mouse instead of trackpad

## Code Examples

```bash
# Read current value
defaults read com.apple.mail SwipeAction

# Disable swipe actions (0)
defaults write com.apple.mail SwipeAction -int 0

# Set to Archive/Delete (default, 1)
defaults write com.apple.mail SwipeAction -int 1

# Set to Mark Read/Unread (2)
defaults write com.apple.mail SwipeAction -int 2

# Set to Flag/Unflag (3)
defaults write com.apple.mail SwipeAction -int 3

# Delete (reset to default)
defaults delete com.apple.mail SwipeAction
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Swipe not working | Ensure trackpad gestures enabled in System Settings |
| Wrong action happening | Check SwipeAction value; verify account settings |
| Can't swipe on some messages | May be server-side restrictions |
| Swipe too sensitive | Adjust trackpad sensitivity in System Settings |
| Action not appearing | Some accounts may not support archive |

## Compatibility

- **macOS**: 10.11+ (El Capitan) for swipe gestures in Mail
- **Note**: Requires trackpad or Magic Mouse
- **Note**: Some email providers don't support archive (use delete instead)
- **Note**: iOS Mail has similar swipe configuration
- **Note**: Works with all account types (iCloud, Gmail, Exchange, etc.)
