# Activity Monitor Dock Icon Type

Control what the Activity Monitor Dock icon displays.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.ActivityMonitor` |
| **Key** | `IconType` |
| **Type** | Integer |
| **Default Value** | `0` (Application Icon) |
| **Our Setting** | `5` (CPU Usage) |

## Description

Controls what the Activity Monitor icon displays in the Dock while the app is running. Instead of showing a static application icon, it can display live system statistics like CPU usage, memory pressure, disk activity, or network throughput. This provides at-a-glance system monitoring without opening the Activity Monitor window.

## Values

| Value | Display | Description |
|-------|---------|-------------|
| `0` | Application Icon | Static Activity Monitor icon |
| `1` | CPU History | Mini CPU history graph |
| `2` | Network Usage | Network in/out indicator |
| `3` | Disk Activity | Disk read/write indicator |
| `4` | Memory Usage | Memory pressure/usage |
| `5` | CPU Usage | Real-time CPU percentage |

## Our Configuration

**Set to**: `5` â€” Show CPU usage percentage for quick monitoring.

### Why CPU Usage (5)?

1. **Most useful** â€” CPU is common bottleneck
2. **Immediate** â€” Know if system is working hard
3. **Clear** â€” Simple percentage display
4. **Debugging** â€” Spot runaway processes
5. **Performance** â€” Monitor impact of tasks

### Other Icon Type Recommendations:

| Use Case | Recommended Type |
|----------|------------------|
| General monitoring | 5 (CPU Usage) |
| Developer/heavy tasks | 1 (CPU History) |
| Network troubleshooting | 2 (Network Usage) |
| Disk performance | 3 (Disk Activity) |
| Memory management | 4 (Memory Usage) |
| Clean dock | 0 (Application Icon) |

## Code Examples

```bash
# Read current value
defaults read com.apple.ActivityMonitor IconType

# Set to CPU usage percentage
defaults write com.apple.ActivityMonitor IconType -int 5

# Set to CPU history graph
defaults write com.apple.ActivityMonitor IconType -int 1

# Set to memory usage
defaults write com.apple.ActivityMonitor IconType -int 4

# Set to network usage
defaults write com.apple.ActivityMonitor IconType -int 2

# Set to disk activity
defaults write com.apple.ActivityMonitor IconType -int 3

# Set to application icon (default)
defaults write com.apple.ActivityMonitor IconType -int 0

# Delete (reset to default)
defaults delete com.apple.ActivityMonitor IconType
```

Quit and reopen Activity Monitor for changes to take effect.

## Related Activity Monitor Settings

```bash
# Dock icon type (this setting)
defaults write com.apple.ActivityMonitor IconType -int 5

# Update frequency (1=Very often, 2=Often, 5=Normally)
defaults write com.apple.ActivityMonitor UpdatePeriod -int 2

# Show all processes
defaults write com.apple.ActivityMonitor ShowCategory -int 100

# Sort column (0=CPU, 1=Memory, etc.)
defaults write com.apple.ActivityMonitor SortColumn -string "CPUUsage"

# Sort direction (0=descending, 1=ascending)
defaults write com.apple.ActivityMonitor SortDirection -int 0
```

## Update Frequency

The dock icon updates based on Activity Monitor's refresh rate:

```bash
# More frequent updates (higher CPU usage)
defaults write com.apple.ActivityMonitor UpdatePeriod -int 1

# Normal updates (default)
defaults write com.apple.ActivityMonitor UpdatePeriod -int 2

# Less frequent updates (saves resources)
defaults write com.apple.ActivityMonitor UpdatePeriod -int 5
```

| Value | Frequency |
|-------|-----------|
| 1 | Very often (1 second) |
| 2 | Often (2 seconds) |
| 5 | Normally (5 seconds) |

## Keep in Dock

To always have system monitoring available:

1. Open Activity Monitor
2. Right-click dock icon
3. Options â†’ Keep in Dock
4. Set preferred IconType
5. Minimize window (icon still updates)

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Icon not updating | Quit and reopen Activity Monitor |
| Wrong display type | Verify setting, restart app |
| Icon stuck at 0% | Check if app is actually running |
| High resource usage | Increase UpdatePeriod value |
| Icon too small to read | Use CPU History instead of percentage |
