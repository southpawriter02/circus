# Xcode Workspace Tab Bar Shows Tab Names

Control whether the tab bar displays tab names in Xcode workspaces.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.dt.Xcode` |
| **Key** | `IDEWorkspaceTabBarShowsTabNames` |
| **Type** | Boolean |
| **Default Value** | `true` |
| **Our Setting** | `true` |

## Description

Controls whether Xcode's workspace tab bar displays the names of open tabs or shows only icons. When enabled, each tab shows its name (file name, scheme name, etc.) making it easier to identify tabs at a glance. When disabled, tabs show only icons, which takes less horizontal space but requires hovering to see the name.

## Values

| Value | Behavior |
|-------|----------|
| `true` | **Show tab names** (default) â€” Full names visible in tab bar |
| `false` | **Icons only** â€” Compact tabs without names |

## Our Configuration

**Set to**: `true` â€” Show tab names for easy file identification.

### Why Show Names?

1. **Identification** â€” Quickly see which file is in each tab
2. **Navigation** â€” Find the right tab without hovering
3. **Context** â€” Know what files are currently open
4. **Fewer mistakes** â€” Less chance of editing wrong file

### Why Hide Names?

- Working with many files simultaneously
- Prefer minimal interface
- Remember files by position
- Small screen with limited space

## Code Examples

```bash
# Read current value
defaults read com.apple.dt.Xcode IDEWorkspaceTabBarShowsTabNames

# Show tab names (default)
defaults write com.apple.dt.Xcode IDEWorkspaceTabBarShowsTabNames -bool true

# Hide tab names (icons only)
defaults write com.apple.dt.Xcode IDEWorkspaceTabBarShowsTabNames -bool false

# Delete (reset to default)
defaults delete com.apple.dt.Xcode IDEWorkspaceTabBarShowsTabNames
```
