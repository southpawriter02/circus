# Xcode Show Minimap

Display a code minimap in the editor for quick navigation.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.dt.Xcode` |
| **Key** | `DVTTextShowMinimap` |
| **Type** | Boolean |
| **Default Value** | `false` |
| **Our Setting** | `true` |

## Description

Controls whether Xcode displays a minimap (miniature overview) of the source code file on the right side of the editor. The minimap provides a bird's-eye view of the entire file, showing code structure, syntax highlighting, and allowing quick navigation by clicking or dragging. This is similar to minimaps found in VS Code, Sublime Text, and other modern editors. The minimap is particularly useful for navigating large files.

## Values

| Value | Behavior |
|-------|----------|
| `false` | **Minimap hidden** (default) — Traditional editor view |
| `true` | **Minimap shown** — Code overview on right side |

## Our Configuration

**Set to**: `true` — Show minimap for better code navigation.

### Why Enable?

1. **Quick navigation** — Jump to any part of file instantly
2. **File overview** — See entire file structure at a glance
3. **Find sections** — Locate MARK: comments visually
4. **Modern workflow** — Standard in VS Code, Sublime, etc.
5. **Long files** — Essential for files with 100+ lines

### Why Disable?

- Small screen/limited horizontal space
- Prefer traditional scrollbar
- Find minimap distracting
- Work with mostly short files
- Need maximum code width

## Code Examples

```bash
# Read current value
defaults read com.apple.dt.Xcode DVTTextShowMinimap

# Enable minimap (recommended)
defaults write com.apple.dt.Xcode DVTTextShowMinimap -bool true

# Disable minimap
defaults write com.apple.dt.Xcode DVTTextShowMinimap -bool false

# Delete (reset to default)
defaults delete com.apple.dt.Xcode DVTTextShowMinimap
```

## Related Settings

```bash
# Show minimap (this setting)
defaults write com.apple.dt.Xcode DVTTextShowMinimap -bool true

# Show line numbers
defaults write com.apple.dt.Xcode DVTTextShowLineNumbers -bool true

# Show code folding ribbon
defaults write com.apple.dt.Xcode DVTTextShowFoldingSidebar -bool true

# Show page guide
defaults write com.apple.dt.Xcode DVTTextShowPageGuide -bool true

# Page guide column position
defaults write com.apple.dt.Xcode DVTTextPageGuideLocation -int 120

# Trim trailing whitespace
defaults write com.apple.dt.Xcode DVTTextEditorTrimTrailingWhitespace -bool true
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Minimap not appearing | Check Editor > Minimap is checked; restart Xcode |
| Minimap takes too much space | Can't resize; consider disabling |
| Not showing syntax colors | Normal; minimap has dimmed colors |
| Click not navigating | Ensure clicking within minimap area |
| Performance issues | Disable for very large files (10K+ lines) |

## Compatibility

- **Xcode**: 11.0+ (introduced minimap feature)
- **macOS**: 10.14.4+ (requires Xcode 11+)
- **Note**: Restart Xcode for defaults change to take effect
- **Note**: Toggle via menu is immediate
- **Note**: Works with all source file types
