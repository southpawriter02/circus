# Notes New Note Body Style

Control the initial formatting style for new notes.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.Notes` |
| **Key** | `NewNoteBodyStyle` |
| **Type** | Integer |
| **Default Value** | `0` |
| **Our Setting** | `0` |

## Description

Controls the initial text formatting style when creating a new note. This determines whether the first line starts as a Title, Heading, Subheading, or plain Body text. The setting affects the visual hierarchy and size of text at the top of new notes, helping establish consistent note structure.

## Values

| Value | Behavior |
|-------|----------|
| `0` | **Title** (default) — Large bold text, best for note names |
| `1` | **Heading** — Medium bold text, secondary emphasis |
| `2` | **Subheading** — Small bold text, tertiary level |
| `3` | **Body** — Regular paragraph text, no emphasis |

## Our Configuration

**Set to**: `0` — Start with Title for named, organized notes.

### Why Use Title?

1. **Organization** — Clear note identification
2. **Note list** — First line becomes note title in list
3. **Hierarchy** — Natural document structure
4. **Searchable** — Titles help find notes later
5. **Professional** — Clean, organized appearance

### Why Use Other Styles?

**Heading (1)**:
- Notes with multiple equal sections
- Quick categorized captures
- Less formal structure needed

**Subheading (2)**:
- Detailed outlines
- Sub-categorized content
- Compact note headers

**Body (3)**:
- Quick scratch notes
- Stream of consciousness
- No formatting desired
- Plain text preference

## Code Examples

```bash
# Read current value
defaults read com.apple.Notes NewNoteBodyStyle

# Set to Title (default)
defaults write com.apple.Notes NewNoteBodyStyle -int 0

# Set to Heading
defaults write com.apple.Notes NewNoteBodyStyle -int 1

# Set to Subheading
defaults write com.apple.Notes NewNoteBodyStyle -int 2

# Set to Body
defaults write com.apple.Notes NewNoteBodyStyle -int 3

# Delete (reset to default)
defaults delete com.apple.Notes NewNoteBodyStyle
```

## Related Settings

```bash
# New note body style (this setting)
defaults write com.apple.Notes NewNoteBodyStyle -int 0

# Default font size (10-36 points)
defaults write com.apple.Notes DefaultFontSize -int 16

# Sort notes order (1=Date Edited, 2=Date Created, 3=Title)
defaults write com.apple.Notes NotesSortOrder -int 1

# Group notes by date
defaults write com.apple.Notes GroupNotesByDate -bool true

# Folders sort order
defaults write com.apple.Notes FoldersSortOrder -int 1
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Setting not applying | Quit and restart Notes |
| Old notes unchanged | Setting only affects new notes |
| Style looks different | May vary with font size setting |
| Want different styles per folder | Not supported; global setting |
| Title too big | Consider Heading (1) or adjust font size |

## Compatibility

- **Notes**: macOS 10.15+ (Catalina) for full formatting
- **Note**: Setting syncs via iCloud
- **Note**: Only affects new notes, not existing
- **Note**: Works with all account types (iCloud, On My Mac)
- **Note**: iOS Notes has similar setting
