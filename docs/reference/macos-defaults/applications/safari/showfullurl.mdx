# Safari Full URL Display

Show the complete URL in the address bar instead of just the domain.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.Safari` |
| **Key** | `ShowFullURLInSmartSearchField` |
| **Type** | Boolean |
| **Default Value** | `false` |
| **Our Setting** | `true` |

## Description

By default, Safari only shows the domain name (e.g., "apple.com") in the address bar for a cleaner look. This setting shows the complete URL including path, query parameters, and anchors â€” essential for developers and security-conscious users.

## Values

| Value | Behavior |
|-------|----------|
| `true` | **Full URL** â€” Show complete address |
| `false` | **Domain only** (default) â€” Simplified display |

## Our Configuration

**Set to**: `true` â€” Always show the full URL.

### Why Show Full URL?

1. **Security** â€” Spot phishing URLs: `apple.com.evil.com` vs `apple.com`
2. **Debugging** â€” See query parameters and paths
3. **Copying** â€” Copy exact URL with one click
4. **Context** â€” Know exactly where you are on a site
5. **Development** â€” Essential for web development work

### Why Apple Hides It

- Cleaner visual design
- Less intimidating for casual users
- Emphasizes domain/security indicator

## Code Examples

```bash
# Show full URL (recommended for developers)
defaults write com.apple.Safari ShowFullURLInSmartSearchField -bool true

# Show domain only (Apple default)
defaults write com.apple.Safari ShowFullURLInSmartSearchField -bool false

# Restart Safari
killall Safari
```
