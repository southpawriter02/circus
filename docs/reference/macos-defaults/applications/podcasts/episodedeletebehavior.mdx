# Podcasts Episode Delete Behavior

Control how played podcast episodes are handled after listening.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.podcasts` |
| **Key** | `EpisodeDeleteBehavior` |
| **Type** | Integer |
| **Default Value** | `0` |
| **Our Setting** | `1` |

## Description

Controls what happens to podcast episodes after you finish listening to them. This setting helps manage storage space by automatically removing played episodes, or keeps them available for re-listening. The behavior can be set globally and overridden on a per-podcast basis. This is particularly useful for managing storage on devices with limited space.

## Values

| Value | Behavior |
|-------|----------|
| `0` | **Manual delete** (default) â€” Keep all episodes until manually deleted |
| `1` | **Delete after 24 hours** â€” Remove played episodes after one day |
| `2` | **Delete after a week** â€” Remove played episodes after seven days |
| `3` | **Delete immediately** â€” Remove as soon as episode finishes |

## Our Configuration

**Set to**: `1` â€” Delete after 24 hours for balanced storage management.

### Why Delete After 24 Hours (1)?

1. **Storage efficient** â€” Prevents accumulation
2. **Grace period** â€” Time to re-listen if needed
3. **Automatic** â€” No manual cleanup required
4. **Balance** â€” Not too aggressive, not too permissive

### Why Keep Manual (0)?

- Want to re-listen to episodes
- Podcasts serve as reference material
- Plenty of storage available
- Prefer manual control

### Why Delete Immediately (3)?

- Very limited storage
- Only listen once
- News/daily podcasts become stale
- Maximum storage savings

## Code Examples

```bash
# Read current value
defaults read com.apple.podcasts EpisodeDeleteBehavior

# Manual delete (keep all, default)
defaults write com.apple.podcasts EpisodeDeleteBehavior -int 0

# Delete after 24 hours
defaults write com.apple.podcasts EpisodeDeleteBehavior -int 1

# Delete after 1 week
defaults write com.apple.podcasts EpisodeDeleteBehavior -int 2

# Delete immediately after playing
defaults write com.apple.podcasts EpisodeDeleteBehavior -int 3

# Delete (reset to default)
defaults delete com.apple.podcasts EpisodeDeleteBehavior
```

## Related Settings

```bash
# Episode delete behavior (this setting)
defaults write com.apple.podcasts EpisodeDeleteBehavior -int 1

# Auto-download new episodes
defaults write com.apple.podcasts downloadNewEpisodes -bool true

# Episode limit per podcast
defaults write com.apple.podcasts episodeLimit -int 10

# Sync podcasts via iCloud
defaults write com.apple.podcasts syncPodcasts -bool true

# Remove played episodes (older setting)
defaults write com.apple.podcasts removePlayedEpisodes -bool true
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Episodes not being deleted | Check per-show settings for overrides |
| Episodes deleted too soon | Change to longer retention (0 or 2) |
| Storage still full | Check episode limit setting as well |
| Want to keep specific episodes | Download to Files or use per-show settings |
| Setting not taking effect | Restart Podcasts app |

## Compatibility

- **macOS**: 10.15+ (Catalina) with Podcasts app
- **Note**: Per-podcast settings can override this global setting
- **Note**: Syncs with iOS Podcasts via iCloud
- **Note**: Downloaded episodes are affected; streamed episodes not stored
- **Note**: Restart Podcasts for terminal changes to apply
