# Numbers Auto-Correction Indicators

Control whether Numbers shows auto-correction indicators in cells.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.iWork.Numbers` |
| **Key** | `TSShowsAutoCorrection` |
| **Type** | Boolean |
| **Default Value** | `true` |
| **Our Setting** | `true` |

## Description

Controls whether Apple Numbers displays visual indicators when text has been auto-corrected in cells. When enabled, auto-corrected text shows a brief highlight or indicator, helping you notice when Numbers has automatically changed your input. This includes spelling corrections, smart quotes, and other automatic text substitutions. The indicators help you verify that auto-corrections are appropriate for your data.

## Values

| Value | Behavior |
|-------|----------|
| `true` | **Show indicators** (default) â€” Highlights auto-corrected text |
| `false` | **Hide indicators** â€” Silent auto-correction |

## Our Configuration

**Set to**: `true` â€” Show auto-correction indicators.

### Why Show Indicators?

1. **Awareness** â€” Know when text was automatically changed
2. **Data integrity** â€” Verify corrections are appropriate
3. **Easy revert** â€” Click to undo unwanted corrections
4. **Technical data** â€” Catch corrections that break codes/IDs
5. **Quality control** â€” Review changes before sharing

### Why Hide Indicators?

- Trust all auto-corrections
- Prefer cleaner cell appearance
- Working with non-critical text
- Fewer visual distractions

## Code Examples

```bash
# Read current value
defaults read com.apple.iWork.Numbers TSShowsAutoCorrection

# Show auto-correction indicators (default)
defaults write com.apple.iWork.Numbers TSShowsAutoCorrection -bool true

# Hide auto-correction indicators
defaults write com.apple.iWork.Numbers TSShowsAutoCorrection -bool false

# Delete (reset to default)
defaults delete com.apple.iWork.Numbers TSShowsAutoCorrection
```

## Related Settings

```bash
# Auto-correction indicators (this setting)
defaults write com.apple.iWork.Numbers TSShowsAutoCorrection -bool true

# Formula warnings
defaults write com.apple.iWork.Numbers TSShowFormulaWarnings -bool true

# Warn before deleting sheets
defaults write com.apple.iWork.Numbers TSWarnBeforeDeletingSheets -bool true

# Show ruler
defaults write com.apple.iWork.Numbers TSShowsRuler -bool true

# Show comments pane
defaults write com.apple.iWork.Numbers TSShowsComments -bool false

# System-wide auto-correction
defaults write NSGlobalDomain NSAutomaticSpellingCorrectionEnabled -bool true
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Indicators not showing | Check Edit > Substitutions > Show Substitutions |
| Too many corrections | Disable specific substitutions in Edit menu |
| Wrong corrections for codes | Disable auto-correct for that cell/selection |
| Setting not persisting | Quit Numbers before changing via defaults |
| Indicators too distracting | Set to false to hide them |
| Can't revert correction | Click the underlined word while editing cell |

## Compatibility

- **macOS**: 10.14+ (Mojave) with Numbers
- **Note**: Same key name used in Pages and Keynote
- **Note**: Respects system-wide substitution preferences
- **Note**: Restart Numbers for terminal changes to apply
- **Note**: Per-cell corrections can be reverted individually
