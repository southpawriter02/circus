# Numbers Tab Key Behavior

Control whether the Tab key inserts a tab character or moves to the next cell in Numbers spreadsheets.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.iWork.Numbers` |
| **Key** | `TSInsertTabInsteadOfMoving` |
| **Type** | Boolean |
| **Default Value** | `false` |
| **Our Setting** | `false` |

## Description

Controls the behavior of the Tab key when editing cells in Numbers. By default, pressing Tab moves the cursor to the next cell (standard spreadsheet behavior). When enabled, Tab inserts a tab character within the cell instead, which is useful for aligning text within multi-line cells but breaks the expected spreadsheet navigation flow.

## Values

| Value | Behavior |
|-------|----------|
| `false` | **Move to next cell** (default) — Standard spreadsheet navigation |
| `true` | **Insert tab character** — Tab character inserted in cell |

## Our Configuration

**Set to**: `false` — Standard Tab-to-move behavior.

### Why Use Tab for Navigation (false)?

1. **Standard behavior** — Matches Excel and Google Sheets
2. **Fast data entry** — Quickly move across columns
3. **User expectation** — Everyone expects Tab to move cells
4. **Keyboard efficiency** — No reaching for arrow keys
5. **Muscle memory** — Works same as other spreadsheets

### Why Insert Tab Characters (true)?

- **Text formatting** — Align text within multi-line cells
- **Import compatibility** — Match tab-delimited data format
- **Specific formatting needs** — Document-style cells

## Code Examples

```bash
# Read current value
defaults read com.apple.iWork.Numbers TSInsertTabInsteadOfMoving

# Tab moves to next cell (default, recommended)
defaults write com.apple.iWork.Numbers TSInsertTabInsteadOfMoving -bool false

# Tab inserts tab character
defaults write com.apple.iWork.Numbers TSInsertTabInsteadOfMoving -bool true

# Delete (reset to default)
defaults delete com.apple.iWork.Numbers TSInsertTabInsteadOfMoving
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Tab not moving to next cell | Disable "Pressing Tab inserts a tab" in Settings |
| Tab inserting characters unexpectedly | Enable Tab navigation in Settings > General |
| Want both behaviors | Use Option+Tab to insert tab when navigation is on |
| Cursor jumps wrong direction | Check if Shift is held (Shift+Tab goes backward) |
| Tab wrapping behavior unexpected | Tab wraps to next row at end of row |
| Setting not persisting | Restart Numbers after changing via defaults |

## Compatibility

- **macOS**: 10.14+ (Mojave) with Numbers
- **Note**: This setting is specific to Numbers
- **Note**: Restart Numbers for terminal changes to apply
- **Note**: Affects editing in table cells only
- **Note**: Option+Tab may insert tab even when navigation is enabled
