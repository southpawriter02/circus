# Photos Delete After Import

Automatically delete photos from device after importing.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.Photos` |
| **Key** | `IPXDefaultsDeleteAfterImport` |
| **Type** | Boolean |
| **Default Value** | `false` |
| **Our Setting** | `false` |

## Description

Controls whether Photos automatically deletes photos and videos from a connected device (iPhone, iPad, camera, SD card) after successfully importing them to your Mac. This can help manage storage on your devices but carries risk of data loss if the import fails or if you need the originals on the device.

## Values

| Value | Behavior |
|-------|----------|
| `true` | **Delete after import** â€” Remove from source device |
| `false` | **Keep originals** (default) â€” Photos remain on device |

## Our Configuration

**Set to**: `false` â€” Keep originals on device after import.

### Why Keep Originals (Disable)?

1. **Data safety** â€” Backup before deleting
2. **Import verification** â€” Check photos imported correctly
3. **Multiple backups** â€” Keep copies in multiple places
4. **Selective deletion** â€” Choose what to delete manually
5. **Avoid accidents** â€” Prevent unintended data loss

### Why Delete After Import?

- Free up device storage immediately
- Automated workflow
- Trust import process
- Device storage is limited
- Regular import routine

## Security Implications

| Risk | Mitigation |
|------|------------|
| **Failed import** | Photos deleted but not imported properly |
| **Corrupted files** | Originals lost if corruption during transfer |
| **Wrong library** | Photos may go to unexpected location |
| **No device backup** | Only copy may be on device |

## Code Examples

```bash
# Read current value
defaults read com.apple.Photos IPXDefaultsDeleteAfterImport

# Disable delete after import (safe, recommended)
defaults write com.apple.Photos IPXDefaultsDeleteAfterImport -bool false

# Enable delete after import (use with caution)
defaults write com.apple.Photos IPXDefaultsDeleteAfterImport -bool true

# Delete (reset to default)
defaults delete com.apple.Photos IPXDefaultsDeleteAfterImport
```

## iCloud Photos Interaction

| Photos Setting | Delete Behavior |
|----------------|-----------------|
| iCloud Photos ON | Device photos likely in iCloud anyway |
| iCloud Photos OFF | Deleting removes only local copy |
| Optimize Storage | Full-res may only be on device |

When iCloud Photos is enabled, this setting is less risky since photos sync to iCloud before import.

## Alternative: Image Capture

For more control over imports:
1. Open **Image Capture** (in /Applications/Utilities)
2. Connect device
3. Import selected photos
4. Choose whether to delete manually

Image Capture doesn't use this Photos preference.

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Photos missing after import | Check Photos library, Recently Deleted |
| Setting not working | Ensure Photos is set as import app |
| Want to undo deletion | Check device's Recently Deleted album |
| Import failed but deleted | Recover from device Recently Deleted |

## Recovery Options

If photos were deleted unexpectedly:

1. **iPhone/iPad Recently Deleted**: Photos app â†’ Albums â†’ Recently Deleted (30 days)
2. **iCloud Photos**: Photos should sync back
3. **Time Machine**: Restore Photos library
4. **Device backup**: Restore from iCloud/iTunes backup
