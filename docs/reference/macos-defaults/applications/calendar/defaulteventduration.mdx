# Calendar Default Event Duration

Set the default length for new events.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.iCal` |
| **Key** | `Default duration in minutes for new event` |
| **Type** | Integer |
| **Default Value** | `60` |
| **Our Setting** | `30` |

## Description

Sets the default duration when creating new events by clicking (not dragging) on the calendar. The event can still be resized after creation.

## Values

| Value | Duration |
|-------|----------|
| `15` | 15 minutes |
| `30` | 30 minutes (our setting) |
| `45` | 45 minutes |
| `60` | 1 hour (default) |
| `90` | 1.5 hours |
| `120` | 2 hours |

## Our Configuration

**Set to**: `30` — Shorter meetings by default.

### Why 30 Minutes?

1. **Parkinson's Law** — Work expands to fill time; shorter = more focused
2. **Meeting efficiency** — Many meetings don't need full hour
3. **Buffer time** — Room between meetings for breaks
4. **Quick catch-ups** — 30 min is enough for most syncs

### The Case for 1 Hour

Some prefer 60 minutes because:
- Complex discussions need time
- Standard business calendar slot
- Buffer for running over

## Code Examples

```bash
# Read current value
defaults read com.apple.iCal "Default duration in minutes for new event"

# 30-minute default events
defaults write com.apple.iCal "Default duration in minutes for new event" -int 30

# 1-hour default (Apple default)
defaults write com.apple.iCal "Default duration in minutes for new event" -int 60

# 15-minute "quick meet" default
defaults write com.apple.iCal "Default duration in minutes for new event" -int 15

# Delete (reset to default)
defaults delete com.apple.iCal "Default duration in minutes for new event"
```

## UI Location

**Calendar** → **Settings** (⌘,) → **General** → **Default event duration**

## Creating Events

| Creation Method | Duration |
|-----------------|----------|
| Click on calendar | Uses this default |
| Drag to create | Uses your drag length |
| Quick Event (⌘N) | Uses this default |

## Related Settings

```bash
# Work hours start/end (for day view)
defaults write com.apple.iCal "first minute of work hours" -int 540  # 9 AM
defaults write com.apple.iCal "last minute of work hours" -int 1080   # 6 PM

# Default alert time
defaults write com.apple.iCal "DefaultEventAlert" -int -900  # 15 min before
```

## Complete Meeting-Friendly Setup

```bash
# Efficient calendar defaults
defaults write com.apple.iCal "Default duration in minutes for new event" -int 30
defaults write com.apple.iCal "DefaultEventAlert" -int -600  # 10 min reminder
defaults write com.apple.iCal "first minute of work hours" -int 540  # 9 AM start
defaults write com.apple.iCal "first day of week" -int 1  # Monday start
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Events still 1 hour by default | Restart Calendar: `killall Calendar` |
| Dragged events wrong length | Drag length overrides default |
| Quick Event uses wrong duration | Setting applies to ⌘N events too |

## Compatibility

- **Calendar**: macOS 10.8+
- **Syncs**: Local preference only
