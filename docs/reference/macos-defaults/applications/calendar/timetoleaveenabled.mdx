# Calendar Time to Leave

Enable travel time notifications that remind you when to leave for appointments.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.iCal` |
| **Key** | `TimeToLeaveEnabled` |
| **Type** | Boolean |
| **Default Value** | `true` |
| **Our Setting** | `true` |

## Description

Controls whether Calendar calculates and displays "Time to Leave" notifications for events with locations. When enabled, Calendar uses the event's location to estimate travel time (based on current traffic conditions for driving) and alerts you when it's time to leave. This integrates with Apple Maps for accurate travel time estimates and considers your current location if Location Services is enabled.

## Values

| Value | Behavior |
|-------|----------|
| `true` | **Time to Leave enabled** (default) â€” Travel time alerts shown |
| `false` | **Time to Leave disabled** â€” No departure reminders |

## Our Configuration

**Set to**: `true` â€” Enable Time to Leave for punctuality.

### Why Enable Time to Leave?

1. **Never late** â€” Accounts for actual travel time
2. **Traffic awareness** â€” Adjusts for real-time conditions
3. **Smart reminders** â€” More useful than fixed alerts
4. **Stress reduction** â€” Know exactly when to leave
5. **Integration** â€” Works with Maps for directions

### Why Disable Time to Leave?

- Working from home (no commute needed)
- Events at same location (unnecessary travel blocks)
- Privacy concerns about location tracking
- Prefer fixed-time reminders
- Living in area with unreliable traffic data

## Code Examples

```bash
# Read current value
defaults read com.apple.iCal TimeToLeaveEnabled

# Enable Time to Leave (default, recommended)
defaults write com.apple.iCal TimeToLeaveEnabled -bool true

# Disable Time to Leave
defaults write com.apple.iCal TimeToLeaveEnabled -bool false

# Delete (reset to default)
defaults delete com.apple.iCal TimeToLeaveEnabled
```

## Related Settings

```bash
# Time to Leave alerts (this setting)
defaults write com.apple.iCal TimeToLeaveEnabled -bool true

# Default event alert time
defaults write com.apple.iCal DefaultEventAlert -int -900

# Default all-day event alert
defaults write com.apple.iCal DefaultAllDayEventAlert -int -39600

# Time zone support
defaults write com.apple.iCal "TimeZone support enabled" -bool true

# Show week numbers
defaults write com.apple.iCal "Show Week Numbers" -bool true

# First day of week
defaults write com.apple.iCal "first day of week" -int 1
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| No Time to Leave alerts | Ensure event has a location address |
| Wrong travel time estimate | Check starting location in event |
| Alerts not appearing | Verify Calendar notifications in System Settings |
| Travel time not shown in calendar | Enable in Calendar > Settings > Alerts |
| Location not found | Use complete address with city/state |
| Traffic not updating | Check internet connection |

## Compatibility

- **macOS**: 10.14+ (Mojave) with Calendar
- **Note**: Requires Location Services for automatic departure point
- **Note**: Traffic data requires internet connection
- **Note**: Works best with complete street addresses
- **Note**: Syncs with iOS Calendar via iCloud
