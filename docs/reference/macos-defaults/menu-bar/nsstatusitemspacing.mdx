# Menu Bar Item Spacing

Control the horizontal spacing between menu bar items.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `NSGlobalDomain` |
| **Key** | `NSStatusItemSpacing` |
| **Type** | Integer |
| **Default Value** | System dependent (~12) |
| **Our Setting** | Not set (system default) |

## Description

Controls the horizontal spacing between menu bar status items (the icons on the right side of the menu bar). Lower values create tighter spacing, allowing more icons to fit in the menu bar. Higher values provide more breathing room between icons for easier clicking and a less crowded appearance. This setting was introduced in macOS Monterey.

## Values

| Value | Behavior |
|-------|----------|
| `6` | **Tight** — Minimal spacing, fits many icons |
| `8` | **Compact** — Reduced spacing |
| `12` | **Default** — Standard macOS spacing |
| `16` | **Spacious** — More room between icons |
| `20` | **Wide** — Maximum breathing room |

## Our Configuration

**Set to**: Not explicitly set — Uses system default spacing.

### Why Keep Default?

1. **Apple's design** — Balanced for most users
2. **Consistent** — Matches expected macOS look
3. **Accessible** — Easy to click
4. **Flexible** — Works for typical setups
5. **No maintenance** — Follows OS updates

### Why Use Tighter Spacing?

- **Many icons** — Fit more menu bar apps
- **Small screen** — Maximize usable space
- **Power user** — Comfortable with dense UI
- **Preference** — Like compact appearance

### Why Use Wider Spacing?

- **Few icons** — No need for density
- **Accessibility** — Larger click targets
- **Aesthetics** — Cleaner, less cluttered
- **Large display** — Space is not at premium

## Code Examples

```bash
# Read current value
defaults read NSGlobalDomain NSStatusItemSpacing

# Set tight spacing (fit more icons)
defaults write NSGlobalDomain NSStatusItemSpacing -int 6

# Set compact spacing
defaults write NSGlobalDomain NSStatusItemSpacing -int 8

# Set spacious spacing
defaults write NSGlobalDomain NSStatusItemSpacing -int 16

# Delete (reset to system default)
defaults delete NSGlobalDomain NSStatusItemSpacing

# Also consider adjusting selection padding
defaults write NSGlobalDomain NSStatusItemSelectionPadding -int 6

# Restart menu bar to apply changes
killall SystemUIServer
```

```bash

# Menu bar item selection padding (clickable area)
defaults write NSGlobalDomain NSStatusItemSelectionPadding -int 6

# Auto-hide menu bar
defaults write NSGlobalDomain _HIHideMenuBar -bool false

# Reduce menu bar transparency
defaults write NSGlobalDomain AppleReduceDesktopTinting -bool false

# Restart menu bar
killall SystemUIServer
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Setting not applying | Run `killall SystemUIServer` |
| Icons still crowded | Try even smaller value (6) |
| Too hard to click | Increase value or use default |
| Third-party icons not affected | Some apps manage their own spacing |
| Setting reverts after reboot | Re-run defaults command |
| Pre-Monterey Mac | Setting may not be supported |

## Compatibility

- **macOS**: 12.0+ (Monterey and later)
- **Note**: May not work on older macOS versions
- **Note**: Requires `killall SystemUIServer` to apply
- **Note**: Some third-party apps may ignore this setting
- **Note**: System icons and third-party icons may behave differently
