# Login Window User List Display

Control whether login shows user list or name/password fields.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.loginwindow` |
| **Key** | `SHOWFULLNAME` |
| **Type** | Boolean |
| **Default Value** | `false` |
| **Our Setting** | `false` |

## Description

Controls whether the login window displays a visual list of user accounts with icons, or requires typing both username and password in text fields. This affects the login experience and has security implications for multi-user Macs.

## Values

| Value | Display | Security |
|-------|---------|----------|
| `false` | User list with icons (default) | Usernames visible |
| `true` | Name + password text fields | Usernames hidden |

## Our Configuration

**Set to**: `false` â€” Show user list.

### Why Show User List?

1. **Single-user Mac** â€” Only one account anyway
2. **Convenience** â€” Click and type password
3. **Touch ID** â€” Works with user list
4. **User comfort** â€” Less to remember
5. **Fast login** â€” Fewer keystrokes

### Why Hide User List (SHOWFULLNAME = true)?

| Scenario | Reason |
|----------|--------|
| Shared/public Mac | Don't reveal usernames |
| High-security environment | Defense in depth |
| Many users | List becomes unwieldy |
| Corporate policy | Compliance requirement |

## Security Analysis

### What User List Reveals

- Valid usernames
- Number of accounts
- User photos (if set)
- Account types (admin badge)

For attackers:
- Half the login credentials (username)
- Social engineering targets
- Account enumeration

### What Text Fields Hide

- Must guess username
- No indication of valid users
- More secure against physical access attacks

## Code Examples

```bash
# Read current value
defaults read com.apple.loginwindow SHOWFULLNAME

# Show user list (default)
defaults write com.apple.loginwindow SHOWFULLNAME -bool false

# Show name/password fields (more secure)
defaults write com.apple.loginwindow SHOWFULLNAME -bool true

# Delete (reset to default)
defaults delete com.apple.loginwindow SHOWFULLNAME
```

## UI Location

**System Settings** â†’ **Users & Groups** â†’ **Login Options** (unlock required)

Setting: "Display login window as"
- List of users
- Name and password

## Combining with Other Security Settings

For maximum login security:
```bash
# Require name AND password
defaults write com.apple.loginwindow SHOWFULLNAME -bool true

# Disable password hints
defaults write com.apple.loginwindow RetriesUntilHint -int 0

# Disable guest account
defaults write com.apple.loginwindow GuestEnabled -bool false

# Disable automatic login
sudo defaults delete com.apple.loginwindow autoLoginUser
```

## Touch ID Consideration

With user list:
- Touch ID works seamlessly
- Finger on sensor â†’ unlocks
- No typing needed

With name/password fields:
- Must enter username first
- Then Touch ID or password
- Slightly slower

## FileVault Impact

With FileVault enabled:
- Login screen shows AFTER disk unlock
- FileVault users may differ from login window
- Pre-boot: password always required

## Remote Login

SSH and screen sharing:
- Unaffected by this setting
- Always require username/password

## Related Settings

# Show input menu (keyboard layouts)
