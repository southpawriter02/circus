# Stage Manager Hide Desktop Items

Control desktop icon visibility in Stage Manager mode.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.WindowManager` |
| **Key** | `HideDesktop` |
| **Type** | Boolean |
| **Default Value** | `false` |
| **Our Setting** | `false` |

## Description

Controls whether desktop items (files, folders, drives, and icons) are visible when Stage Manager is active. When enabled, the desktop appears clean with no items, providing a distraction-free workspace. Desktop items remain accessible through Finder but don't appear on the desktop background.

## Values

| Value | Behavior |
|-------|----------|
| `false` | **Show desktop items** (default) — Icons visible on desktop |
| `true` | **Hide desktop items** — Clean, empty desktop |

## Our Configuration

**Set to**: `false` — Keep desktop items visible.

### Why Show Desktop Items?

1. **Accessibility** — Quick access to files
2. **Visual reference** — See active projects
3. **Drag and drop** — Easy file management
4. **Familiarity** — Traditional desktop experience
5. **Temporary files** — Quick download/screenshot access

### Why Hide Desktop Items?

- Clean, minimal aesthetic
- Distraction-free environment
- Presentation mode
- Wallpaper appreciation
- Force organized workflow

## Code Examples

```bash
# Read current value
defaults read com.apple.WindowManager HideDesktop

# Hide desktop items
defaults write com.apple.WindowManager HideDesktop -bool true

# Show desktop items (default)
defaults write com.apple.WindowManager HideDesktop -bool false

# Delete (reset to default)
defaults delete com.apple.WindowManager HideDesktop
```

Changes take effect immediately.

## Related Settings

```bash

# Enable Stage Manager
defaults write com.apple.WindowManager GloballyEnabled -bool true

# Window grouping behavior
defaults write com.apple.WindowManager AppWindowGroupingBehavior -int 0

# Show recent apps strip
defaults write com.apple.WindowManager AutoHide -bool false

# Hide widgets in Stage Manager
defaults write com.apple.WindowManager StandardHideWidgets -bool false
```

## Finder Desktop Settings

Note: This is separate from Finder's desktop item settings:

```bash
# Finder: Show hard disks on desktop
defaults write com.apple.finder ShowHardDrivesOnDesktop -bool true

# Finder: Show external disks on desktop
defaults write com.apple.finder ShowExternalHardDrivesOnDesktop -bool true

# Finder: Show removable media on desktop
defaults write com.apple.finder ShowRemovableMediaOnDesktop -bool true

# Finder: Show connected servers on desktop
defaults write com.apple.finder ShowMountedServersOnDesktop -bool true
```

These Finder settings control what CAN appear on desktop.
The Stage Manager HideDesktop controls whether desktop items are VISIBLE.

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Desktop still shows items | Restart Stage Manager (toggle) |
| Can't find files | Use Finder > Go > Desktop |
| Drives not appearing | Check Finder preferences |
| Setting not applying | Verify Stage Manager is enabled |
| Items reappear | Check if Stage Manager was disabled |

## Behavior Notes

- **External drives**: Still mount, just not visible on desktop
- **Screenshots**: Still saved to desktop (if that's your setting)
- **Downloads**: Still saved to desktop (if that's your setting)
- **Drag to desktop**: May not work when hidden
- **Click desktop**: Shows all windows, not items

## Compatibility

- **macOS**: 13.0+ (Ventura)
- **Note**: Only applies when Stage Manager is enabled
- **Note**: Takes effect immediately
- **Note**: Desktop folder contents unaffected
