# Stage Manager Hide Widgets

Control whether widgets are hidden when Stage Manager is active.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.WindowManager` |
| **Key** | `StageManagerHideWidgets` |
| **Type** | Boolean |
| **Default Value** | `false` |
| **Our Setting** | `false` |

## Description

Controls whether desktop widgets are hidden when Stage Manager is enabled. When Stage Manager is active, this setting determines if widgets remain visible on the desktop or are hidden to provide a cleaner workspace. Widgets can include weather, calendar, reminders, and other information displays that were introduced in macOS Sonoma.

## Values

| Value | Behavior |
|-------|----------|
| `false` | **Widgets shown** (default) — Widgets visible on desktop |
| `true` | **Widgets hidden** — Widgets hidden when Stage Manager active |

## Our Configuration

**Set to**: `false` — Keep widgets visible for at-a-glance information.

### Why Show Widgets?

1. **Information access** — Weather, calendar, reminders visible
2. **Quick reference** — Glance at info without switching apps
3. **Desktop utility** — Desktop remains useful
4. **Notification Center alternative** — Info without extra gesture
5. **macOS Sonoma feature** — Use widgets as designed

### Why Hide Widgets?

- Prefer minimal, clean desktop
- Distraction-free workspace
- Widgets not personally useful
- Better screenshot appearance
- Focus on active windows only

## Code Examples

```bash
# Read current value
defaults read com.apple.WindowManager StageManagerHideWidgets

# Show widgets (default)
defaults write com.apple.WindowManager StageManagerHideWidgets -bool false

# Hide widgets
defaults write com.apple.WindowManager StageManagerHideWidgets -bool true

# Delete (reset to default)
defaults delete com.apple.WindowManager StageManagerHideWidgets
```

## Related Settings

```bash
# Hide widgets (this setting)
defaults write com.apple.WindowManager StageManagerHideWidgets -bool false

# Enable Stage Manager globally
defaults write com.apple.WindowManager GloballyEnabled -bool true

# Auto-hide recent apps strip
defaults write com.apple.WindowManager AutoHide -bool false

# Hide desktop icons
defaults write com.apple.WindowManager HideDesktop -bool false

# Click wallpaper to show desktop
defaults write com.apple.WindowManager EnableStandardClickToShowDesktop -bool true

# App window grouping
defaults write com.apple.WindowManager AppWindowGroupingBehavior -int 0
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Setting has no effect | Stage Manager must be enabled first |
| Widgets not appearing | Add widgets: click desktop > Edit Widgets |
| Can't find widgets | Access via Notification Center when hidden |
| Setting resets | Check if managed by MDM or profile |
| Widgets on wrong monitor | Widget placement is per-display |

## Compatibility

- **macOS**: 14.0+ (Sonoma) — Widgets on desktop introduced
- **macOS**: 13.0+ (Ventura) — Stage Manager introduced
- **Note**: Widgets feature requires macOS Sonoma or later
- **Note**: Setting only affects Stage Manager mode
- **Note**: On Ventura, widgets are in Notification Center only
