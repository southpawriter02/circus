# Click Desktop to Show Desktop

Control whether clicking on the desktop background reveals the desktop.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.WindowManager` |
| **Key** | `EnableStandardClickToShowDesktop` |
| **Type** | Boolean |
| **Default Value** | `true` |
| **Our Setting** | `true` |

## Description

Controls whether clicking on the desktop wallpaper (an area not covered by windows) reveals the desktop by moving all windows aside. When enabled, clicking on any visible desktop area temporarily hides all windows, allowing access to desktop files and folders. This behavior was changed in macOS Sonoma and can be disabled for users who prefer the previous behavior where clicking the desktop did nothing special.

## Values

| Value | Behavior |
|-------|----------|
| `true` | **Click reveals desktop** (default in Sonoma+) â€” Windows move aside |
| `false` | **Click does nothing** â€” Traditional behavior; windows stay put |

## Our Configuration

**Set to**: `true` â€” Enable click-to-reveal-desktop for quick access.

### Why Enable?

1. **Quick access** â€” Fast way to see desktop files
2. **iOS consistency** â€” Matches iPad Stage Manager behavior
3. **Intuitive** â€” Natural expectation when clicking desktop
4. **Convenient** â€” No keyboard shortcut needed

### Why Disable?

- Prefer previous macOS behavior
- Accidental clicks reveal desktop unexpectedly
- Always use keyboard shortcuts
- Find behavior disorienting

## Code Examples

```bash
# Read current value
defaults read com.apple.WindowManager EnableStandardClickToShowDesktop

# Enable click to show desktop (default in Sonoma+)
defaults write com.apple.WindowManager EnableStandardClickToShowDesktop -bool true

# Disable click to show desktop (traditional behavior)
defaults write com.apple.WindowManager EnableStandardClickToShowDesktop -bool false

# Delete (reset to default)
defaults delete com.apple.WindowManager EnableStandardClickToShowDesktop
```

## Related Settings

```bash
# Click to show desktop (this setting)
defaults write com.apple.WindowManager EnableStandardClickToShowDesktop -bool true

# Stage Manager enabled
defaults write com.apple.WindowManager GloballyEnabled -bool false

# Stage Manager hide desktop items
defaults write com.apple.WindowManager HideDesktop -bool false

# Stage Manager hide widgets
defaults write com.apple.WindowManager StageManagerHideWidgets -bool false

# Stage Manager auto-hide strip
defaults write com.apple.WindowManager AutoHide -bool false
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Click does nothing | Enable setting or use keyboard shortcut |
| Accidental reveals | Disable setting; use shortcuts instead |
| Stage Manager overrides | Stage Manager has its own click behavior |
| Setting not persisting | Restart Dock: killall Dock |
| Different behavior than expected | Check Stage Manager status |

## Compatibility

- **macOS**: 14+ (Sonoma) for this specific setting
- **Note**: In earlier macOS, clicking desktop never revealed desktop
- **Note**: Stage Manager (macOS 13+) introduced similar behavior first
- **Note**: Interacts with Stage Manager settings
- **Note**: Restart may be required for changes
