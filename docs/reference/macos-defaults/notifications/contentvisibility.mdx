# Notification Content Visibility

Control when notification content previews are displayed.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.ncprefs` |
| **Key** | `content_visibility` |
| **Type** | Integer |
| **Default Value** | `1` |
| **Our Setting** | `2` |

## Description

Controls when the actual content of notifications (message text, email subjects, etc.) is displayed in notification banners and the Notification Center. This is a privacy feature that prevents others from reading your notifications when your Mac is unattended or visible to others. You can choose to always show previews, never show them, or only show them when your Mac is unlocked.

## Values

| Value | Behavior |
|-------|----------|
| `0` | **Never show previews** â€” Only app name and generic alert |
| `1` | **Always show previews** (default) â€” Full content visible always |
| `2` | **When unlocked only** â€” Content shown only when logged in |

## Our Configuration

**Set to**: `2` â€” Show previews only when unlocked for privacy.

### Why "When Unlocked" (Recommended)?

1. **Privacy** â€” Passersby can't read your messages
2. **Security** â€” Sensitive information protected
3. **Convenience** â€” Full content when you're using Mac
4. **Balance** â€” Best of both worlds
5. **Professional** â€” Appropriate for office environments

### Why "Always"?

- Single-user Mac in private location
- Quick notification triage important
- No privacy concerns
- Personal preference for convenience

### Why "Never"?

- High-security environment
- Shared display situations
- Maximum privacy required
- Screen frequently visible to others

## Code Examples

```bash
# Read current value
defaults read com.apple.ncprefs content_visibility

# Never show previews (most private)
defaults write com.apple.ncprefs content_visibility -int 0

# Always show previews (default)
defaults write com.apple.ncprefs content_visibility -int 1

# When unlocked only (recommended)
defaults write com.apple.ncprefs content_visibility -int 2

# Delete (reset to default)
defaults delete com.apple.ncprefs content_visibility
```

## Related Settings

```bash
# Content visibility (this setting)
defaults write com.apple.ncprefs content_visibility -int 2

# Banner display time
defaults write com.apple.notificationcenterui bannerTime -int 5

# Show widgets on desktop
defaults write com.apple.WindowManager ShowOnDesktop -bool true

# Focus/DND (use System Settings or Control Center)
# These are stored in CloudKit, not easily set via defaults
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Setting not applying | Restart NotificationCenter: `killall NotificationCenter` |
| One app ignoring setting | Check per-app override in Notifications settings |
| Previews still showing locked | Verify setting is 2 not 1; restart required |
| Can't change setting | May be managed by MDM profile |
| Want different behavior per app | Use per-app settings in System Settings |

## Compatibility

- **macOS**: 10.15+ (Catalina) for current behavior
- **Note**: Older macOS versions may have different key names
- **Note**: Per-app settings stored in separate database
- **Note**: Focus modes can override this setting
- **Note**: MDM can enforce notification policies
