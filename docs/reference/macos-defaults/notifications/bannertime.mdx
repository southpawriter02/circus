# Notification Banner Display Time

Control how long notification banners stay visible on screen.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.notificationcenterui` |
| **Key** | `bannerTime` |
| **Type** | Integer (seconds) |
| **Default Value** | `5` |
| **Our Setting** | `3` |

## Description

Controls the duration that notification banners remain visible in the upper-right corner of the screen before automatically dismissing. Shorter times reduce distraction, while longer times give more time to read and interact with notifications. This affects all banner-style notifications system-wide, though individual apps can be configured to use alerts (which require manual dismissal) instead.

## Values

| Value | Duration |
|-------|----------|
| `2` | 2 seconds â€” Very brief |
| `3` | 3 seconds â€” Quick glance |
| `5` | 5 seconds (default) â€” Standard |
| `7` | 7 seconds â€” Extended |
| `10` | 10 seconds â€” Long display |
| `15` | 15 seconds â€” Maximum practical |

## Our Configuration

**Set to**: `3` â€” Brief banners to minimize distraction.

### Why Use Shorter Banner Time?

1. **Less distraction** â€” Banners disappear quickly
2. **Focus** â€” Less interruption to workflow
3. **Clean screen** â€” Banners don't linger
4. **Efficiency** â€” Glance and continue working
5. **Notification Center** â€” Full details available there

### Why Use Longer Banner Time?

- Accessibility needs (more reading time)
- Important notifications shouldn't be missed
- Slower reading speed
- Want time to click/interact
- Less frequently at computer

## Notification Interaction

Even with short banner time, you can:

| Action | How |
|--------|-----|
| Pause dismissal | Hover mouse over banner |
| Quick actions | Click banner or use buttons |
| View later | Check Notification Center |
| Dismiss immediately | Swipe right or click X |

## Code Examples

```bash
# Read current value
defaults read com.apple.notificationcenterui bannerTime

# Set to 3 seconds (quick)
defaults write com.apple.notificationcenterui bannerTime -int 3

# Set to 5 seconds (default)
defaults write com.apple.notificationcenterui bannerTime -int 5

# Set to 10 seconds (extended)
defaults write com.apple.notificationcenterui bannerTime -int 10

# Delete (reset to default)
defaults delete com.apple.notificationcenterui bannerTime

# Restart Notification Center to apply
killall NotificationCenter
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Setting not applying | Run `killall NotificationCenter` |
| Banners still long | Check if app uses Alerts instead of Banners |
| No banners at all | Check notification permissions for app |
| Can't interact in time | Increase bannerTime or switch app to Alerts |
| Setting resets | May be managed by MDM profile |

## Recommended Durations by Use Case

| Use Case | Recommended Duration |
|----------|---------------------|
| Power users | 2-3 seconds |
| General use | 5 seconds (default) |
| Accessibility | 7-10 seconds |
| Mission-critical apps | Use Alerts instead |
| Minimal distraction | 2 seconds |

## Accessibility Note

For users who need more time to read notifications:

```bash
# Extended banner time for accessibility
defaults write com.apple.notificationcenterui bannerTime -int 10

# Or switch critical apps to Alert style in:
# System Settings â†’ Notifications â†’ [App] â†’ Alert style: Alerts
```

## Compatibility

- **macOS**: 10.8+ (Mountain Lion) with Notification Center
- **Note**: Only affects Banner-style notifications
- **Note**: Alerts always require manual dismissal
- **Note**: May require NotificationCenter restart
