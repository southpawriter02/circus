# Reduce Desktop Tinting

Control transparency and tinting effects in the macOS interface.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `NSGlobalDomain` |
| **Key** | `AppleReduceDesktopTinting` |
| **Type** | Boolean |
| **Default Value** | `false` |
| **Our Setting** | `false` |

## Description

Controls the transparency and vibrancy effects throughout the macOS interface. When enabled, this reduces the translucency of windows, sidebars, menu bars, and other interface elements, making them more solid and opaque. This is primarily an accessibility feature that helps users who have difficulty reading text or distinguishing interface elements against transparent backgrounds. It also affects desktop tinting, where windows pick up colors from the desktop wallpaper.

## Values

| Value | Behavior |
|-------|----------|
| `false` | **Normal transparency** (default) — Full vibrancy and translucency effects |
| `true` | **Reduced transparency** — More solid, opaque interface elements |

## Our Configuration

**Set to**: `false` — Keep normal transparency for standard visual design.

### Why Keep Normal Transparency (false)?

1. **Modern aesthetic** — Intended macOS design language
2. **Visual depth** — Transparency provides UI layering cues
3. **Default experience** — What most users expect
4. **Wallpaper integration** — Desktop feels cohesive
5. **No accessibility need** — For users without visual difficulties

### Why Enable Reduced Transparency (true)?

- **Accessibility** — Vision difficulties require higher contrast
- **Readability** — Text easier to read on solid backgrounds
- **Preference** — Some users prefer solid interface
- **Performance** — Minimal GPU benefit on older systems
- **Screenshots** — Consistent appearance for documentation

## Code Examples

```bash
# Read current value
defaults read NSGlobalDomain AppleReduceDesktopTinting

# Keep normal transparency (default)
defaults write NSGlobalDomain AppleReduceDesktopTinting -bool false

# Enable reduced transparency (accessibility)
defaults write NSGlobalDomain AppleReduceDesktopTinting -bool true

# Delete (reset to default)
defaults delete NSGlobalDomain AppleReduceDesktopTinting
```

Changes may require logout or restart to fully apply.

## Effects on Interface Elements

| Element | Normal (false) | Reduced (true) |
|---------|----------------|----------------|
| Menu bar | Translucent, picks up wallpaper | Solid gray |
| Finder sidebar | Translucent with tinting | Solid background |
| Dock background | Translucent | More opaque |
| Notification Center | Vibrancy effect | Solid background |
| Sheets/dialogs | Translucent edges | Solid edges |
| Spotlight search | Translucent background | Solid background |

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Change not visible | Logout and log back in |
| Menu bar still transparent | Restart SystemUIServer: `killall SystemUIServer` |
| Only affects some apps | Some third-party apps don't respect this setting |
| Want partial transparency | Not possible - it's all or nothing |
| Setting reverts | Check for conflicting accessibility settings |

## Compatibility

- **macOS**: 10.10+ (Yosemite, when transparency was introduced)
- **Note**: Effect varies slightly between macOS versions
- **Note**: May require logout for complete application
- **Note**: Some third-party apps may not respect this setting
- **Note**: Part of macOS accessibility framework
