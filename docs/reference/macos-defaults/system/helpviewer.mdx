# Help Viewer Window Mode

Control whether Help Viewer floats above other windows.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.helpviewer` |
| **Key** | `DevMode` |
| **Type** | Boolean |
| **Default Value** | `false` |
| **Our Setting** | `true` |

## Description

Controls whether the Help Viewer window behaves like a normal window or floats above all other windows. By default, Help Viewer floats on top, which can be intrusive when following instructions. When DevMode is enabled, Help Viewer acts like a regular window that can go behind other windows, making it easier to work alongside help documentation.

## Values

| Value | Behavior |
|-------|----------|
| `false` | **Floating window** (default) â€” Always on top |
| `true` | **Normal window** â€” Can go behind other windows |

## Our Configuration

**Set to**: `true` â€” Help Viewer as normal window for better workflow.

### Why Enable DevMode (Normal Window)?

1. **Usability** â€” Work alongside help
2. **Multitasking** â€” Switch windows naturally
3. **Less intrusive** â€” Not always blocking
4. **Side-by-side** â€” Compare help with app
5. **Professional** â€” Standard window behavior

### Why Keep Floating (Default)?

- Help always visible for reference
- Don't lose Help behind windows
- Single monitor setup
- Prefer modal help experience
- Following step-by-step guides

## Code Examples

```bash
# Read current value
defaults read com.apple.helpviewer DevMode

# Enable normal window mode (recommended)
defaults write com.apple.helpviewer DevMode -bool true

# Keep floating window (default)
defaults write com.apple.helpviewer DevMode -bool false

# Delete (reset to default)
defaults delete com.apple.helpviewer DevMode
```

Changes take effect immediately for new Help windows.

## Why "DevMode"?

The key is named "DevMode" because:
- Developers often need to reference documentation
- Side-by-side coding and docs is common workflow
- Floating windows are problematic for development
- Named when Apple added the option for developers
