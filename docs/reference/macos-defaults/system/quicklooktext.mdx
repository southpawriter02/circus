# QuickLook Text Selection

Enable text selection and copying in QuickLook previews.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.finder` |
| **Key** | `QLEnableTextSelection` |
| **Type** | Boolean |
| **Default Value** | `false` |
| **Our Setting** | `true` |

## Description

Controls whether you can select and copy text from QuickLook previews. By default, QuickLook shows a read-only preview of files. When enabled, you can select text in QuickLook previews of text files, PDFs, and other documents, then copy it to the clipboard without opening the file in a full application.

## Values

| Value | Behavior |
|-------|----------|
| `false` | **Read-only preview** (default) â€” Cannot select text |
| `true` | **Text selection enabled** â€” Select and copy text |

## Our Configuration

**Set to**: `true` â€” Enable text selection in QuickLook.

### Why Enable Text Selection?

1. **Speed** â€” Copy without opening apps
2. **Efficiency** â€” Stay in Finder workflow
3. **Convenience** â€” Quick text extraction
4. **Resource savings** â€” Don't launch heavy apps
5. **Productivity** â€” Common need for many users

### Why Keep Disabled (Default)?

- Prefer viewing-only preview
- Accidental text selection is annoying
- QuickLook already has "Open with" button
- Don't need to copy from previews
- Minimal use case

## Code Examples

```bash
# Read current value
defaults read com.apple.finder QLEnableTextSelection

# Enable text selection
defaults write com.apple.finder QLEnableTextSelection -bool true

# Disable text selection (default)
defaults write com.apple.finder QLEnableTextSelection -bool false

# Delete (reset to default)
defaults delete com.apple.finder QLEnableTextSelection

# Restart Finder to apply
killall Finder
```

## UI Location

There is no GUI for this setting. Use the Terminal command.

To use QuickLook:
1. Select a file in Finder
2. Press **Space** bar
3. Preview appears
4. Press **Space** again to close

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Can't select text | Run `killall Finder` after setting |
| Selection works in some files only | File type may not support it |
| Text appears but won't select | PDF may be image-based (scanned) |
| Setting doesn't persist | Verify with `defaults read` |

## OCR for Scanned Documents

Scanned PDFs (image-based) don't support text selection even with this setting. Use:
- Preview â†’ Tools â†’ Text Selection (if OCR layer exists)
- Third-party OCR software
- macOS Live Text (for images)
