# Dock Magnification

Enable icon magnification when hovering over the Dock.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.dock` |
| **Keys** | `magnification`, `largesize` |
| **Type** | Boolean, Integer |
| **Default Value** | `false`, `128` |
| **Our Setting** | `false`, `64` |

## Description

Dock magnification causes icons to enlarge as you hover your cursor over them, creating a "fish-eye lens" effect. This helps identify apps when using a small Dock size but can feel distracting to some users. The `largesize` key controls how large icons become when magnified.

## Values

### Magnification Toggle

| Value | Behavior |
|-------|----------|
| `true` | **Enabled** â€” Icons enlarge on hover |
| `false` | **Disabled** (default) â€” Icons stay constant size |

### Large Size (pixels)

| Value | Size |
|-------|------|
| `16` | Tiny magnification |
| `64` | Subtle magnification |
| `128` | Medium magnification (default when enabled) |
| `256` | Maximum magnification |

## Our Configuration

**Magnification**: `false` â€” Consistent Dock size
**Large Size**: `64` â€” Subtle if enabled

### Why Disable Magnification?

1. **Predictability** â€” Dock doesn't shift around
2. **Click accuracy** â€” Icons don't move as you approach
3. **Clean aesthetic** â€” Minimal visual effects
4. **Performance** â€” Slightly less GPU work (negligible)

### When to Enable Magnification

- Small Dock tile size (hard to see icons)
- Many apps in Dock (need visual distinction)
- Preference for dynamic interfaces
- Accessibility (larger targets on hover)

## Code Examples

```bash
# Read current values
defaults read com.apple.dock magnification
defaults read com.apple.dock largesize

# Disable magnification (recommended)
defaults write com.apple.dock magnification -bool false
killall Dock

# Enable magnification with large icons
defaults write com.apple.dock magnification -bool true
defaults write com.apple.dock largesize -int 128
killall Dock

# Enable subtle magnification
defaults write com.apple.dock magnification -bool true
defaults write com.apple.dock largesize -int 64
killall Dock

# Delete (reset to defaults)
defaults delete com.apple.dock magnification
defaults delete com.apple.dock largesize
killall Dock
```

## Size Relationships

For best results, set `largesize` larger than `tilesize`:

```bash
# Small Dock with large magnification
defaults write com.apple.dock tilesize -int 36
defaults write com.apple.dock magnification -bool true
defaults write com.apple.dock largesize -int 128
killall Dock

# Medium Dock with subtle magnification
defaults write com.apple.dock tilesize -int 48
defaults write com.apple.dock magnification -bool true
defaults write com.apple.dock largesize -int 64
killall Dock
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Magnification not working | Ensure `magnification` is true |
| Too much/little magnification | Adjust `largesize` value |
| Dock position shifts | Normal behavior with magnification |
| Setting not applying | Run `killall Dock` |

## Compatibility

- **macOS**: All versions
- **Applies to**: Main Dock only
- **Note**: Touch Bar Dock (if present) is separate
