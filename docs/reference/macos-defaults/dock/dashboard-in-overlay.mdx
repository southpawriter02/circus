# Dock Dashboard in Overlay

Control whether Dashboard appears as an overlay or as a separate Space.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.dock` |
| **Key** | `dashboard-in-overlay` |
| **Type** | Boolean |
| **Default Value** | `true` |
| **Our Setting** | `true` |

## Description

Controls how Dashboard appears when activated. Dashboard was a feature in macOS that provided quick access to widgets like weather, stocks, calculator, and notes. When set to overlay mode, Dashboard appears over your current workspace as a translucent layer. When set to Space mode, Dashboard is a separate desktop space that you swipe to. Note: Dashboard was deprecated in macOS Catalina (10.15) and removed in later versions.

## Values

| Value | Behavior |
|-------|----------|
| `true` | **Overlay mode** (default) — Dashboard appears over current screen |
| `false` | **Space mode** — Dashboard is a separate desktop space |

## Our Configuration

**Set to**: `true` — Dashboard as overlay (when Dashboard was available).

### Why Use Overlay Mode (true)?

1. **Quick access** — No space switching needed
2. **Context preservation** — See desktop behind widgets
3. **Fast dismiss** — Press Escape to close
4. **Less intrusive** — Doesn't take up a Space
5. **Original design** — How Dashboard was intended

### Why Use Space Mode (false)?

- **Persistent view** — Keep widgets visible while working
- **Better organization** — Dedicated widget space
- **Swipe workflow** — Natural gesture navigation
- **Full screen** — More room for widgets

## Code Examples

```bash
# Read current value
defaults read com.apple.dock dashboard-in-overlay

# Set Dashboard as overlay (default)
defaults write com.apple.dock dashboard-in-overlay -bool true

# Set Dashboard as separate Space
defaults write com.apple.dock dashboard-in-overlay -bool false

# Completely disable Dashboard (macOS Mojave and earlier)
defaults write com.apple.dashboard mcx-disabled -bool true

# Re-enable Dashboard
defaults write com.apple.dashboard mcx-disabled -bool false

# Delete (reset to default)
defaults delete com.apple.dock dashboard-in-overlay

# Restart Dock to apply changes
killall Dock
```

## Related Settings

```bash
# Dashboard display mode (this setting)
defaults write com.apple.dock dashboard-in-overlay -bool true

# Disable Dashboard entirely
defaults write com.apple.dashboard mcx-disabled -bool true

# Don't rearrange Spaces based on most recent use
defaults write com.apple.dock mru-spaces -bool false

# Group windows by application in Mission Control
defaults write com.apple.dock expose-group-apps -bool true

# Displays have separate Spaces
defaults write com.apple.spaces spans-displays -bool false

# Hot corner for Dashboard (bottom-left corner)
defaults write com.apple.dock wvous-bl-corner -int 7

# Apply changes
killall Dock
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Dashboard not appearing | Dashboard removed in macOS Catalina+; use widgets instead |
| Setting not applying | Run `killall Dock` after changing |
| Dashboard disabled | Check `mcx-disabled` setting |
| No Dashboard option in settings | macOS Catalina+ removed Dashboard |
| Want widgets in newer macOS | Use Notification Center or Desktop widgets |
| Hot corner not working | Set hot corner specifically for Dashboard |

## Compatibility

- **macOS 10.4-10.14**: Dashboard available
- **macOS 10.7+**: Space mode option added (this setting)
- **macOS 10.15 (Catalina)**: Dashboard deprecated
- **macOS 11.0+ (Big Sur)**: Dashboard removed entirely
- **Note**: This setting has no effect on macOS Catalina and later
- **Note**: Use Notification Center widgets as modern replacement

**File**: [`defaults/interface/dock.sh`](../../../../../defaults/interface/dock.sh)
