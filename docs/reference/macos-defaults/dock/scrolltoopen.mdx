# Dock Scroll to Open

Use scroll gesture to open folder stacks in Dock.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.dock` |
| **Key** | `scroll-to-open` |
| **Type** | Boolean |
| **Default Value** | `true` |
| **Our Setting** | `true` |

## Description

Controls whether you can use a scroll gesture (mouse wheel or two-finger trackpad scroll) to quickly expand folder stacks in the Dock without clicking. When enabled, hovering over a folder stack and scrolling opens it immediately, providing faster access to files. This is particularly useful for power users who frequently access files from Dock stacks.

## Values

| Value | Behavior |
|-------|----------|
| `true` | **Enabled** (default) â€” Scroll to open stacks |
| `false` | **Disabled** â€” Click only to open stacks |

## Our Configuration

**Set to**: `true` â€” Scroll to open for faster access.

### Why Enable Scroll to Open?

1. **Speed** â€” Faster than clicking
2. **Natural** â€” Intuitive scroll gesture
3. **Efficiency** â€” One less click
4. **Power user** â€” Quick file access
5. **Trackpad friendly** â€” Works well with gestures

### Why Disable Scroll to Open?

- Accidental openings while scrolling near Dock
- Prefer explicit click to open
- Precision mouse users
- Don't use folder stacks
- Prefer predictable behavior

## Code Examples

```bash
# Read current value
defaults read com.apple.dock scroll-to-open

# Enable scroll to open (default)
defaults write com.apple.dock scroll-to-open -bool true

# Disable scroll to open
defaults write com.apple.dock scroll-to-open -bool false

# Delete (reset to default)
defaults delete com.apple.dock scroll-to-open

# Restart Dock to apply changes
killall Dock
```

## Related Settings

```bash
# Scroll to open (this setting)
defaults write com.apple.dock scroll-to-open -bool true

# Spring loading (hover opens folders when dragging)
defaults write com.apple.dock enable-spring-load-actions-on-all-items -bool true

# Mouse-over highlight stack items
defaults write com.apple.dock mouse-over-hilite-stack -bool true

# Show recent apps in Dock
defaults write com.apple.dock show-recents -bool false

# Dock auto-hide
defaults write com.apple.dock autohide -bool false
```

## Stack Configuration Options

Right-click a folder stack to configure:

| Option | Description |
|--------|-------------|
| Sort by | Name, Date Added, Date Modified, Date Created, Kind |
| Display as | Stack (preview), Folder (folder icon) |
| View content as | Fan, Grid, List, Automatic |

## Fan vs Grid vs List

| View | Best For |
|------|----------|
| Fan | Few items (<10), quick visual scan |
| Grid | Many items, organized browsing |
| List | Detailed info, sorted access |
| Automatic | Let macOS decide based on count |

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Scroll not opening stacks | Verify setting, `killall Dock` |
| Opens too easily | Disable setting to require clicks |
| Doesn't work with mouse | Check mouse scroll settings |
| Stack shows wrong view | Right-click, set "View content as" |
| Files not appearing | Refresh Dock, check folder contents |

## Compatibility

- **macOS**: 10.5+ (Leopard)
- **Note**: Restart Dock for changes to take effect
- **Note**: Works with mouse wheel and trackpad scroll
- **Note**: Only applies to folder stacks, not apps
