# show-recents

Hide recent applications section in the Dock.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.dock` |
| **Key** | `show-recents` |
| **Type** | Boolean |
| **Default Value** | `true` |
| **Our Setting** | `false` |

## Description

Controls whether the Dock shows recently used applications in a separate section. Disabling creates a cleaner, more predictable Dock layout.

## Values

| Value | Behavior |
|-------|----------|
| `true` | **Show recents** (default) — Recent apps appear on right side |
| `false` | **Hide recents** — Cleaner Dock, no transient icons |

## Our Configuration

**Set to**: `false` — Clean, predictable Dock without transient icons.

## Code Examples

```bash
# Read current value
defaults read com.apple.dock show-recents

# Hide recent applications
defaults write com.apple.dock show-recents -bool false

# Show recent applications (Apple default)
defaults write com.apple.dock show-recents -bool true

# Delete (reset to default)
defaults delete com.apple.dock show-recents

# Restart Dock to apply
killall Dock
```

## UI Location

**System Settings** → **Desktop & Dock** → **Show recent applications in Dock**

## Why Hide Recents?

- Dock stays consistent across sessions
- No unexpected icons appearing
- Cleaner, less cluttered appearance
- More predictable icon positions

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Recents still showing | Restart Dock: `killall Dock` |
| Setting resets after reboot | Verify with `defaults read` |
| Want some recents | Use Launchpad or Spotlight instead |

## Compatibility

- **Dock**: macOS 10.14+
- **Note**: Recents section added in Mojave
