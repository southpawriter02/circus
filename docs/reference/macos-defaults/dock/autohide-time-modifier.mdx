# Dock Auto-Hide Animation Speed

Control the duration of the Dock show/hide animation.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.dock` |
| **Key** | `autohide-time-modifier` |
| **Type** | Float (seconds) |
| **Default Value** | `0.5` |
| **Our Setting** | `0.3` |

## Description

Controls how fast the Dock slides in and out of view when auto-hide is enabled. The value represents the animation duration in seconds. Lower values make the Dock appear and disappear faster, while higher values create a slower, more gradual animation. Setting to `0` makes the Dock appear instantly with no animation.

## Values

| Value | Speed | Description |
|-------|-------|-------------|
| `0` | Instant | No animation, appears/disappears immediately |
| `0.1` | Very Fast | Quick snap animation |
| `0.3` | Fast | Snappy but visible animation |
| `0.5` | Normal | Default macOS animation speed |
| `1.0` | Slow | Leisurely animation |
| `2.0` | Very Slow | Dramatic, slow reveal |

## Our Configuration

**Set to**: `0.3` â€” Faster animation for quicker Dock access.

### Why Use Faster Animation?

1. **Productivity** â€” Less waiting for Dock to appear
2. **Responsive feel** â€” System feels snappier
3. **Workflow** â€” Quick app switching
4. **Focus** â€” Less distraction from slow animations
5. **Screen space** â€” Dock hidden more of the time

### Why Use Slower Animation?

- Visual aesthetics preference
- Easier to track Dock position
- Matches default macOS behavior
- Gentler on the eyes
- Presentation/demo purposes

## Prerequisites

This setting only affects the Dock when auto-hide is enabled:

```bash
# Enable Dock auto-hide first
defaults write com.apple.dock autohide -bool true

# Then set animation speed
defaults write com.apple.dock autohide-time-modifier -float 0.3

# Apply changes
killall Dock
```

## Code Examples

```bash
# Read current value
defaults read com.apple.dock autohide-time-modifier

# Set instant (no animation)
defaults write com.apple.dock autohide-time-modifier -float 0

# Set fast animation (0.3 seconds)
defaults write com.apple.dock autohide-time-modifier -float 0.3

# Set default animation (0.5 seconds)
defaults write com.apple.dock autohide-time-modifier -float 0.5

# Set slow animation (1.0 second)
defaults write com.apple.dock autohide-time-modifier -float 1.0

# Delete (reset to default)
defaults delete com.apple.dock autohide-time-modifier

# Apply changes
killall Dock
```

## Instant Dock Configuration

For maximum responsiveness:

```bash
# No delay waiting
defaults write com.apple.dock autohide-delay -float 0

# No animation
defaults write com.apple.dock autohide-time-modifier -float 0

# Apply
killall Dock
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Setting has no effect | Ensure auto-hide is enabled first |
| Dock still slow | Check autohide-delay value too |
| Animation jerky | Value may be too low, try 0.2-0.3 |
| Setting resets | Re-run killall Dock after setting |
| Dock doesn't appear | Move cursor firmly to screen edge |

## Performance Considerations

| Value | CPU Impact | Visual Impact |
|-------|------------|---------------|
| 0 | Minimal | Instant pop |
| 0.1-0.3 | Low | Smooth but quick |
| 0.5 | Normal | Default experience |
| 1.0+ | Higher | May feel sluggish |

Lower values use less animation frames, reducing CPU usage slightly.

## Compatibility

- **macOS**: 10.8+ (Mountain Lion)
- **Requires**: Auto-hide enabled (`autohide -bool true`)
- **Note**: Requires `killall Dock` to apply
- **Note**: Not available in System Settings GUI
