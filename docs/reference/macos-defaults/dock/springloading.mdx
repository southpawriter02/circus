# Dock Spring Loading

Open folders and apps when dragging items over them.

## Overview

| Property | Value |
|----------|-------|
| **Domain** | `com.apple.dock` |
| **Key** | `enable-spring-load-actions-on-all-items` |
| **Type** | Boolean |
| **Default Value** | `true` |
| **Our Setting** | `true` |

## Description

Spring loading allows you to drag a file over a folder or application in the Dock and have it automatically open after a brief hover. This enables quick file operations without manually opening applications or navigating to folders first. Holding a dragged file over a Dock item "springs" it open.

## Values

| Value | Behavior |
|-------|----------|
| `false` | **Disabled** — Dragging over Dock items does not spring-load them |
| `true` | **Enabled** (default) — Hovering while dragging opens items |

## Our Configuration

**Set to**: `true` — Spring loading enabled for efficient file management.

### Why Enable Spring Loading?

1. **Efficiency** — Move files without opening windows first
2. **One-handed** — Complete operation with single drag
3. **Deep navigation** — Access nested folders quickly
4. **App integration** — Drop files directly into apps
5. **Natural workflow** — Intuitive drag-and-drop behavior

### Why Disable Spring Loading?

- Accidental folder/app opening
- Prefer explicit actions
- Slower computers may lag
- Learning curve confusion
- Specific workflow requirements

## Code Examples

```bash
# Read current value
defaults read com.apple.dock enable-spring-load-actions-on-all-items

# Enable spring loading (default)
defaults write com.apple.dock enable-spring-load-actions-on-all-items -bool true

# Disable spring loading
defaults write com.apple.dock enable-spring-load-actions-on-all-items -bool false

# Adjust spring loading delay (in seconds)
defaults write NSGlobalDomain com.apple.springing.delay -float 0.5

# Enable spring loading in Finder (related)
defaults write NSGlobalDomain com.apple.springing.enabled -bool true

# Delete (reset to default)
defaults delete com.apple.dock enable-spring-load-actions-on-all-items

# Restart Dock to apply changes
killall Dock
```

## Related Settings

```bash
# Dock spring loading (this setting)
defaults write com.apple.dock enable-spring-load-actions-on-all-items -bool true

# Global spring loading
defaults write NSGlobalDomain com.apple.springing.enabled -bool true

# Spring loading delay (seconds)
defaults write NSGlobalDomain com.apple.springing.delay -float 0.5

# Finder spring loading (separate setting)
defaults write com.apple.finder SpringingEnabled -bool true

# Dock icon size
defaults write com.apple.dock tilesize -int 48

# Show recent apps in Dock
defaults write com.apple.dock show-recents -bool false
```

## Spring Loading Delay Values

| Delay | Description | Best For |
|-------|-------------|----------|
| 0.2s | Very short | Power users, fast workflows |
| 0.5s | Default | Most users |
| 0.75s | Longer | Preventing accidents |
| 1.0s+ | Very long | Careful, deliberate use |

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Spring loading not working | Restart Dock (`killall Dock`) |
| Too slow to open | Reduce spring loading delay |
| Opens too quickly | Increase spring loading delay |
| Doesn't work on all items | Some items may not support it |
| Works in Dock but not Finder | Check Finder spring loading setting |

## Compatibility
